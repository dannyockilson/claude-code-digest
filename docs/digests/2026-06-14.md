# Claude Code Weekly Intelligence Digest

Claude Code shipped two major model releases this week—Fable 5 (Mythos-class) and comprehensive SDK support for the new models—while simultaneously addressing critical infrastructure issues including session deadlocks and authentication failures. ## 🚨 Breaking Changes & Action Required

### [Claude Code GitHub Actions v1.0](https://code.claude.com/docs/en/github-actions)
Claude Code GitHub Actions moved to GA with mandatory workflow syntax changes. Action version must update to `@v1`, `mode` config removed (auto-detected), `direct_prompt` renamed to `prompt`, and CLI options moved to `claude_args` parameter. Beta users must upgrade workflow files immediately or workflows will fail.

**Why it matters:** Production CI/CD pipelines using beta syntax will break without manual migration.

---

### [MCP Python SDK v2.0.0a1 Alpha](https://github.com/modelcontextprotocol/python-sdk/releases/tag/v2.0.0a1)
Protocol fundamentally shifts from stateful bidirectional to stateless request/response architecture. Handlers now constructor parameters (not decorators), return values no longer auto-wrapped, snake_case field names enforced, and FastMCP renamed to MCPServer. Alphas ship through June 2026 with breaking changes each release; stable v2.0.0 on 2026-07-27.

**Action required:** Add upper bound constraint `mcp>=1.27,<2` to dependencies NOW to prevent automatic v2 resolution breaking production deployments.

---

### [Claude Code v2.1.170](https://github.com/anthropics/claude-code/releases/tag/v2.1.170)
Sessions launched from VS Code integrated terminal failed to save transcripts and weren't resumable with `--resume` flag due to inherited environment variables. Fixed in v2.1.170 but impacts session persistence for users launching from IDE terminals.

**Why it matters:** Affects workflow reliability for developers using Claude Code within VS Code instead of standalone terminals.

---

### [Claude Code v2.1.174](https://github.com/anthropics/claude-code/releases/tag/v2.1.174)
Fixed Bedrock GovCloud regions deriving wrong inference profile prefix (`global` instead of `us-gov`), causing 400 errors on model ID requests. Background sessions now use clean inheritance instead of inheriting provider env vars from shell that started daemon.

**Why it matters:** Critical fix for AWS GovCloud deployments and multi-tenant environments relying on environment isolation.
