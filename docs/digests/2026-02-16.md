## 🚨 Breaking Changes & Action Required

### claude-code-action v1.0.49: Git Permission Syntax Update
[Release Notes](https://github.com/anthropics/claude-code-action/releases/tag/v1.0.49)

GitHub Actions integration now requires updating git permission syntax from deprecated `:*` to `*` wildcard in workflow files. Additionally, the CI MCP server now gracefully skips installation when `actions:read` permission is unavailable, preventing failures with restricted GitHub Apps. PR checkout branch conflict resolution also fixed.

**Why it matters:** Existing workflows using custom git permissions will break silently—audit your `.github/workflows/` files now.

## 🆕 New Releases

### Claude Code v2.1.41: CLI Auth Overhaul + ARM64 Windows
[Release Notes](https://github.com/anthropics/claude-code/releases/tag/v2.1.41)

New `claude auth login/status/logout` CLI subcommands provide structured authentication management. Critical fixes include AWS auth timeout capped at 3 minutes (previously hung indefinitely), FileReadTool now handles FIFOs/stdin blocking correctly, and Windows ARM64 gets native binary support. Quality-of-life improvements: `/rename` auto-generates session names, `@-mentions` resolve anchor fragments (`@README.md#installation`).

**Why it matters:** Resolves the most common auth debugging pain points; Windows ARM64 support expands Copilot+ PC compatibility.

---

### Claude Code v2.1.39: Terminal Stability Fixes
[Release Notes](https://github.com/anthropics/claude-code/releases/tag/v2.1.39)

Fatal errors now surface correctly instead of silent failures, process hanging after session close eliminated, and terminal rendering performance improved. Character loss at screen boundaries and blank lines in verbose transcripts also fixed.

## 📦 SDK & API Updates

### Claude Agent SDK v0.1.36: Structured Thinking Control
[Release Notes](https://github.com/anthropics/claude-agent-sdk-python/releases/tag/v0.1.36)

Introduces `ThinkingConfig` with four effort levels (`low`/`medium`/`high`/`max`) for fine-grained extended thinking control in `ClaudeAgentOptions`. The new `thinking` field deprecates `max_thinking_tokens`. Bundles Claude CLI v2.1.42.

**Why it matters:** First SDK-level API for tuning reasoning intensity—critical for balancing latency vs. solution quality in production agents.

## 🔌 MCP Ecosystem

### Pare: Token-Optimized Dev Tool MCP Servers
[Repository](https://github.com/Dave-London/Pare)

MCP servers for git, test runners, npm, and Docker designed around tokens-per-operation metrics instead of traditional API ergonomics. Structured output schemas for git operations (diff/log/status), npm scripts, and Docker commands eliminate unstructured command output bloat. Example: `git log` returns actionable deltas (changed files only) instead of streaming full commit history.

**Why it matters:** Directly addresses the agent workflow tax—raw command output consumes context budget exponentially; Pare's filters deliver 10-100x token efficiency.

## 📄 Docs Changes

### Claude Code Overview: Terminal CLI Now Primary Interface
[Docs](https://code.claude.com/docs/en/overview)

Documentation rewritten to emphasize Terminal CLI as primary interface with native auto-updating installs (macOS/Linux/WSL) and Windows support (PowerShell/CMD). New workflow details: `/teleport` for web→terminal session transfer, `/desktop` for visual diffs, agent teams for parallel tasks, and MCP integration for external tools (Jira/Slack/Google Drive). CLAUDE.md project config now documented as core feature for coding standards, custom slash commands (`/review-pr`, `/deploy-staging`), and pre/post-action hooks. Multi-surface parity emphasized across Terminal, VS Code, JetBrains, Desktop, Web, CI/CD (GitHub Actions/GitLab), and Slack.

**Why it matters:** Signals architectural shift—Terminal CLI moving from "advanced option" to canonical interface.
