## 🚨 Breaking Changes & Action Required

### [Claude Code v2.1.113: Native Binary Migration](https://github.com/anthropics/claude-code/releases/tag/v2.1.113)
The CLI now uses native platform binaries instead of bundled JavaScript. Security hardening includes bash deny rules that now match wrapped commands (env/sudo/watch), `/private` paths flagged as dangerous removal targets, and `find -exec` no longer auto-approved. Added granular domain blocking via `sandbox.network.deniedDomains` setting alongside existing `allowedDomains` wildcards. 40+ fixes spanning MCP timeout handling, markdown rendering, and subagent transcript visibility.

**Why it matters:** Performance gains are substantial, but the security changes may break existing approval patterns—audit your automation scripts.

### [Agent SDK TypeScript v0.2.113: Environment Variable Breaking Change](https://github.com/anthropics/claude-agent-sdk-typescript/releases/tag/v0.2.113)
`options.env` now **replaces** instead of overlays `process.env` for CLI subprocesses—you must explicitly spread `process.env` if adding variables. New alpha `sessionStore` API mirrors transcripts to external storage. OpenTelemetry trace context now propagates to CLI subprocesses for distributed tracing (requires `claude-agent-sdk[otel]`).

**Why it matters:** Silent environment isolation breaks existing configs relying on inherited env vars—verify before deploying.

### [GitHub Actions GA: Beta Migration Required](https://code.claude.com/docs/en/github-actions)
Action upgraded from `@beta` to `@v1` with unified parameter structure: replace `direct_prompt` → `prompt`, migrate `max_turns/model/custom_instructions` to `claude_args`, remove `mode` configuration. Now supports Opus 4.7 (`claude_args: --model claude-opus-4-7`) and cloud providers (AWS Bedrock, Google Vertex AI with Workload Identity Federation).

**Why it matters:** Beta workflows will break—update YAML configs immediately to avoid CI failures.

## 🆕 New Releases

### [Claude Opus 4.7 + xhigh Effort Level (v2.1.111)](https://github.com/anthropics/claude-code/releases/tag/v2.1.111)
New middle ground between high/max effort via `/effort` interactive slider, auto mode for Max subscribers. `/ultrareview` command runs comprehensive multi-agent code review on current branch or GitHub PR. Windows PowerShell tool rolling out (opt-in via `CLAUDE_CODE_USE_POWERSHELL_TOOL`). Read-only bash commands with globs and `cd <dir> &&` patterns skip permission prompts. Plan files auto-named from prompts.

**Why it matters:** xhigh fills the intelligence/speed gap for common workflows; `/ultrareview` replaces manual PR review loops.

### [Prompt Caching 1-Hour TTL (v2.1.108)](https://github.com/anthropics/claude-code/releases/tag/v2.1.108)
Bedrock/Vertex/Foundry now support 1-hour TTL via `ENABLE_PROMPT_CACHING_1H` (deprecates `ENABLE_PROMPT_CACHING_1H_BEDROCK`). Added `/recap` feature to surface prior context when resuming sessions (auto-invoked or manual). Slash commands (`/init`, `/review`, `/security-review`) now discoverable via Skill tool.

**Why it matters:** Reduces API costs for long sessions by 50-90% depending on cache hit rates.

## 📦 SDK & API Updates

### [Python SDK: Subagent Transcript Inspection (v0.1.60)](https://github.com/anthropics/claude-agent-sdk-python/releases/tag/v0.1.60)
New `list_subagents()` and `get_subagent_messages()` helpers enable direct querying of spawned subagent message chains—critical for debugging multi-agent workflows. W3C distributed tracing propagates `TRACEPARENT`/`TRACESTATE` to CLI subprocesses. **Breaking:** `setting_sources=[]` now correctly disables all filesystem settings (verify existing code).

**Why it matters:** Manual log parsing no longer required for multi-agent debugging.

### [TypeScript SDK: Opus 4.7 + Token Budgets (v0.90.0)](https://github.com/anthropics/anthropic-sdk-typescript/releases/tag/sdk-v0.90.0)
Added `claude-opus-4-7` model support, token budgets API for request throttling/cost control, and user profiles API for managing user-specific configurations. Sonnet/Opus 4 models marked deprecated in v0.89.0—migrate to newer versions.

**Why it matters:** Token budgets enable production cost guardrails; deprecation timeline unclear.

### [Agent SDK: Startup() and WarmQuery Exposed (v0.2.111)](https://github.com/anthropics/claude-agent-sdk-typescript/releases/tag/v0.2.111)
`startup()` and `WarmQuery` now in public TypeScript API. `mcp_set_servers` control request gains per-tool `permission_policy` for remote HTTP/SSE servers (granular tool access control).

**Why it matters:** Fine-grained MCP security for multi-tenant or restricted environments.

## 📄 Docs Changes

### [Overview: Multi-Surface Workflows](https://code.claude.com/docs/en/overview)
Docs now emphasize native CLI installs (macOS/Linux/WSL/Windows), Homebrew, WinGet with auto-updates. MCP integration, agent teams, and Agent SDK documented as core features. Remote Control (mobile continuation), Dispatch (async task handoff), `claude --teleport` (web→terminal), and `/desktop` (visual review) added for cross-platform routing. Routines (Anthropic-managed scheduling) and Channels (webhook integration) enable CI automation.

**Why it matters:** Documents production-grade handoff patterns missing from initial releases.

### [Hooks Guide: Auto-Approval Patterns](https://code.claude.com/docs/en/hooks-guide)
Complete lifecycle reference added: SessionStart, PreToolUse, PostToolUse, PermissionRequest, ConfigChange, CwdChanged, FileChanged, Notification. PermissionRequest hooks can skip dialogs by writing JSON decisions to stdout. Practical patterns: block file edits (protected files via exit code 2), re-inject context after compaction, audit config changes.

**Why it matters:** Auto-approval workflows now officially supported—reduces manual interruptions in trusted environments.

### [MCP: Dynamic Tool Updates](https://code.claude.com/docs/en/mcp)
Highlights `list_changed` notifications for dynamic tool updates, automatic reconnection with exponential backoff for HTTP/SSE servers, and plugin-provided MCP servers auto-loading from `.mcp.json`. Three scopes (local, project, user) control MCP availability; project-scoped servers in version-controlled `.mcp.json` enable team collaboration. Environment variable expansion (`${VAR}`, `${VAR:-default}`) now supported.

**Why it matters:** Team-shared MCP configs reduce onboarding friction for new developers.
