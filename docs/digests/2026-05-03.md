## 🚨 Breaking Changes & Action Required

### Claude Code GitHub Actions v1.0 GA
[Documentation](https://code.claude.com/docs/en/github-actions) | Relevance: 8/10

GitHub Actions integration reaches GA with breaking changes requiring workflow file updates. Change `@beta` to `@v1`, remove `mode` config (now auto-detected), replace `direct_prompt` with `prompt`, and migrate CLI options to `claude_args` parameter. New enterprise features include Amazon Bedrock and Google Vertex AI support via Workload Identity Federation for data residency control.

**Why it matters:** Beta users will see workflow failures until migration is complete; enterprise teams gain multi-cloud deployment options.

---

### MCP Python SDK Dependency Floor Raised
[Release v0.1.70](https://github.com/anthropics/claude-agent-sdk-python/releases/tag/v0.1.70) | Relevance: 8/10

`claude-agent-sdk-python` v0.1.70 bumps `mcp` dependency floor to >=1.19.0 to fix silent tool result dropping. Older MCP versions caused agents to receive validation errors instead of actual tool outputs, breaking agentic workflows. Also resolved `Trio` nursery corruption when canceling `query()` iteration with stderr enabled.

**Why it matters:** Agent integrations silently failing on tool execution will require immediate dependency update.

---

### MCP Specification Security Requirements
[Specification](https://modelcontextprotocol.io/specification) | Relevance: 8/10

MCP spec now mandates explicit user consent for data access/tool execution, data privacy controls, and LLM sampling approval. Formalizes trust model: Hosts (LLM apps) initiate connections to Servers (context/tool providers) via JSON-RPC with capability negotiation. Tool descriptions treated as untrusted unless from trusted servers.

**Why it matters:** Claude Code MCP server implementations must implement consent flows and access controls to remain spec-compliant.

## 🆕 New Releases

### Claude Code v2.1.126
[Release Notes](https://github.com/anthropics/claude-code/releases/tag/v2.1.126) | Relevance: 8/10

`/model` picker now respects Anthropic-compatible gateways via `ANTHROPIC_BASE_URL` for multi-provider deployments. New `claude project purge [path]` command with `--dry-run` and `--interactive` flags cleanly deletes project state (transcripts, tasks, file history). `--dangerously-skip-permissions` bypasses `.claude/`, `.git/`, `.vscode/`, and shell config prompts. OAuth login now works in WSL2, SSH, and containers when browser callback can't reach localhost. Includes 15+ bug fixes for stream timeouts, image scaling, and credential race conditions.

**Why it matters:** Unblocks enterprise multi-cloud setups and improves dev workflow ergonomics for power users.

---

### Claude Code v2.1.122
[Release Notes](https://github.com/anthropics/claude-code/releases/tag/v2.1.122) | Relevance: 7/10

Added `ANTHROPIC_BEDROCK_SERVICE_TIER` env var to select Bedrock tier (default/flex/priority) for AWS cost/latency control. `/resume` now finds sessions by PR URL across GitHub/GitLab/Bitbucket. Fixed `/branch` fork creation failures with tool_use errors on rewound timelines. Multiple observability improvements: OpenTelemetry emits numeric attributes correctly, added `claude_code.at_mention` log events, fixed Bedrock/Vertex AI 400 errors for proxy users.

**Why it matters:** AWS users gain fine-grained inference cost control; PR-based session discovery speeds up context retrieval.

---

### Claude Code v2.1.121
[Release Notes](https://github.com/anthropics/claude-code/releases/tag/v2.1.121) | Relevance: 8/10

`alwaysLoad` MCP option skips tool-search deferral for instant tool availability. `claude plugin prune` removes orphaned dependencies; `plugin uninstall --prune` cascades deletion. PostToolUse hooks now support `updatedToolOutput` across all tools (previously MCP-only). Fixed critical memory leaks: unbounded growth processing images, `/usage` leaking ~2GB, and long-running tool failures.

**Why it matters:** Memory leak fixes are critical for long-running sessions; universal tool output customization enables broader SDK/extension patterns.
