# Claude Code Weekly Intelligence Digest

Five Claude Code releases (v2.1.77–v2.1.81) shipped major improvements including 64k–128k token limits for Opus/Sonnet 4.6, a new `--bare` flag for headless automation, rate limit tracking in the statusline, and critical fixes for OAuth token refresh races and parallel tool result restoration. ## 🚨 Breaking Changes & Action Required

### **GitHub Actions Migration Required**
**[Claude Code GitHub Actions](https://code.claude.com/docs/en/github-actions)** — GA release requires action upgrade from `@beta` to `@v1` with configuration changes: `direct_prompt` renamed to `prompt`, mode auto-detection removed (specify explicitly), CLI options consolidated into `claude_args`. Update your workflows before the beta channel is deprecated.

**Why it matters:** Failing to migrate will break existing CI/CD pipelines when beta is sunset.

---

### **Security Fix: Hooks Can No Longer Bypass Permission Rules**
**[Claude Code v2.1.77](https://github.com/anthropics/claude-code/releases/tag/v2.1.77)** — PreToolUse hooks now respect `deny` permission rules and enterprise managed settings. Previously, hooks could circumvent sandbox isolation. If your workflows relied on PreToolUse bypassing permissions, they will now fail.

**Why it matters:** Ensures sandbox integrity but may break custom automation that depended on the bypass behavior.

---

### **Parallel Tool Results Restoration Fixed**
**[Claude Agent SDK TypeScript v0.2.80](https://github.com/anthropics/claude-agent-sdk-typescript/releases/tag/v0.2.80)** — Critical fix: `getSessionMessages()` now correctly returns all tool_use/tool_result pairs in parallel scenarios. Previously dropped results were breaking agent state management. Update immediately if you rely on parallel tool calls.

**Why it matters:** State corruption in multi-tool workflows is silently resolved; test your agents after upgrading.

## 🆕 New Releases

### **Claude Code v2.1.81: Headless Automation & OAuth Improvements**
**[Release Notes](https://github.com/anthropics/claude-code/releases/tag/v2.1.81)** — New `--bare` flag strips hooks/LSP/plugin sync for scripted `-p` calls, enabling clean headless automation (requires explicit API key, no OAuth/keychain). Fixed critical OAuth token refresh race causing repeated re-auth in concurrent sessions. MCP OAuth now supports Client ID Metadata Document (CIMD/SEP-991) for broader server compatibility.

**Why it matters:** Bare mode unblocks CI/CD and scripting use cases where full IDE features create noise or failures.

---

### **Claude Code v2.1.80: Rate Limit Visibility & Inline Plugins**
**[Release Notes](https://github.com/anthropics/claude-code/releases/tag/v2.1.80)** — New `rate_limits` field in statusline displays Claude.ai 5-hour/7-day usage windows with percentage and reset times. `source: 'settings'` allows inline plugin declarations in settings.json without marketplace. Preview `--channels` flag enables MCP servers to push messages into sessions.

**Why it matters:** Statusline rate limits prevent surprise quota exhaustion during intensive sessions; inline plugins simplify local/custom workflows.

---

### **Claude Code v2.1.79: Console Auth & Subprocess Hang Fix**
**[Release Notes](https://github.com/anthropics/claude-code/releases/tag/v2.1.79)** — Added `--console` flag to `claude auth login` for Anthropic Console (API billing) authentication. Fixed critical subprocess hang in `claude -p` mode when stdin isn't piped (e.g., Python `subprocess.run`). VSCode extension gains `/remote-control` for session bridging to claude.ai/code.

**Why it matters:** Subprocess fix unblocks Python/Node.js automation; Console auth separates billing from user credentials.

---

### **Claude Code v2.1.77: Token Limit Increase & Faster Resume**
**[Release Notes](https://github.com/anthropics/claude-code/releases/tag/v2.1.77)** — Token limits increased to 64k default/128k max for Opus/Sonnet 4.6, enabling longer code generation runs. `/copy N` now copies specific assistant responses by index; `/fork` renamed to `/branch` (alias retained). 45% faster `--resume` on large sessions with ~100-150MB less peak memory.

**Why it matters:** 2x token limit dramatically extends single-session scope; resume speedup improves daily workflow velocity.
