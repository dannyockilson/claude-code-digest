## 🚨 Breaking Changes & Action Required

### Claude Code GitHub Actions v1.0 GA
[Documentation](https://code.claude.com/docs/en/github-actions)
Claude Code GitHub Actions exits beta with mandatory workflow updates: change `@beta` to `@v1`, remove `mode` config (auto-detected), rename `direct_prompt` to `prompt`, and move CLI options to new `claude_args` parameter. Opus 4.8 now available as model option; setup simplified via `/install-github-app` CLI command or manual GitHub app + `ANTHROPIC_API_KEY` secret configuration.

**Why it matters:** Existing CI/CD workflows will break without updates—plan migration before next workflow run.

---

### anthropic-sdk-typescript Middleware Execution Order Fix
[v0.102.0](https://github.com/anthropics/anthropic-sdk-typescript/releases/tag/sdk-v0.102.0) | [bedrock-sdk-v0.30.1](https://github.com/anthropics/anthropic-sdk-typescript/releases/tag/bedrock-sdk-v0.30.1) | [aws-sdk-v0.4.1](https://github.com/anthropics/anthropic-sdk-typescript/releases/tag/aws-sdk-v0.4.1)
Middleware now runs BEFORE request signing across all TypeScript SDK variants (core, Bedrock, AWS). Custom middleware modifying headers or auth must be verified for compatibility—previously signed requests were being modified post-signature, causing silent auth failures.

**Why it matters:** Production integrations using custom middleware may have been failing silently; update and test immediately.

---

### Agent SDK Default Tool Behavior Change
[v0.3.162](https://github.com/anthropics/claude-agent-sdk-typescript/releases/tag/v0.3.162)
Agent SDK sessions now default to embedded Bash `find`/`grep` instead of dedicated Grep/Glob tools on native builds. To preserve tool interception for security/observability, explicitly register tools in `tools` option or `allowedTools`.

**Why it matters:** Hooks and permission systems relying on Grep/Glob tool interception will stop working without config changes.

## 🆕 New Releases

### Claude Code v2.1.166
[Release Notes](https://github.com/anthropics/claude-code/releases/tag/v2.1.166)
Fallback model chains now support up to 3 models in sequence on overload/unavailability, reducing manual recovery workflows. Glob pattern support added to deny rules (`*` denies all tools) with startup warnings for unknown MCP tools. Thinking can now be disabled via `MAX_THINKING_TOKENS=0`, `--thinking disabled`, or per-model toggle for API models that think by default. API error handling hardened with retries on unexpected non-retryable errors using fallback model; cross-session permission request blocking bug fixed.

**Why it matters:** Production reliability improvements—fallback chains eliminate manual intervention during model outages.

---

### Claude Code v2.1.163
[Release Notes](https://github.com/anthropics/claude-code/releases/tag/v2.1.163)
Managed version controls (`requiredMinimumVersion`/`requiredMaximumVersion`) enforce org policies; new `/plugin list` command with filters and `/btw` markdown copy shortcut for clipboard preservation. Critical fixes: resolved `claude -p` hanging on backgrounded commands, bash failures under bazel/EDR workflows, Windows OneDrive read-only issues, and hook condition regressions on `$()` subshell patterns.

**Why it matters:** Org-wide version enforcement enables centralized policy compliance for enterprise deployments.

### Claude Code v2.1.162
[Release Notes](https://github.com/anthropics/claude-code/releases/tag/v2.1.162)
`claude agents --json` exposes `waitingFor` field for debugging blocked sessions (permission prompts, etc.)—critical for automation monitoring. Fixed Windows path handling: permission rules with backslashes and case variants now match correctly; Read deny rules hide files from Glob/Grep. MCP `timeout` values <1000ms now properly ignored instead of flooring to 1s watchdog. UX polish: slash commands autocomplete fills instead of auto-runs, Remote Control shows persistent footer.

**Why it matters:** Windows users can finally use permission rules reliably; `waitingFor` field unblocks automated session monitoring.

### Claude Code v2.1.161
[Release Notes](https://github.com/anthropics/claude-code/releases/tag/v2.1.161)
Parallel tool calls now fail independently—broken bash commands no longer cancel concurrent operations. `claude agents` output shows `done/total` counters with longest-running item in peek. Fixed regression: `--resume` correctly finds sessions in non-git directories. Security fix: `claude mcp` commands redact credentials when listing/adding MCPs; `${VAR}` expansion skipped.

**Why it matters:** Multi-tool workflows are significantly more reliable—one failure doesn't cascade anymore.

### Claude Code v2.1.160
[Release Notes](https://github.com/anthropics/claude-code/releases/tag/v2.1.160)
Security-first release adds prompts before writing to shell startup files (.zshenv, .bash_login) and git config to prevent unintended command execution. acceptEdits mode prompts before modifying build-tool configs (.npmrc, .yarnrc, bunfig.toml, .bazelrc). Single-file grep commands now satisfy read-before-edit checks. WSL copy-on-select uses PowerShell interop instead of OSC 52. Background sessions retain chat history on restore/overnight retire.

**Why it matters:** Prevents supply-chain and command-execution attacks via accidental build config modifications.
