## 🚨 Breaking Changes & Action Required

### [Claude Code v2.1.136: MCP OAuth & Server Persistence Fixes](https://github.com/anthropics/claude-code/releases/tag/v2.1.136)
Fixed concurrent MCP OAuth refresh token corruption preventing daily re-auth loops in multi-server setups. Patched critical bug where `.mcp.json`/plugin servers silently disappeared after `/clear` across all IDEs. Security fix: `Edit(...)` allow rules no longer bypass file write blocks in plan mode. **Why it matters:** Eliminates enterprise authentication failures and data loss in persistent MCP workflows.

### [Claude Code v2.1.133: `worktree.baseRef` Default Changed](https://github.com/anthropics/claude-code/releases/tag/v2.1.133)
**BREAKING:** `EnterWorktree` now defaults to `origin/<default>` instead of local `HEAD`. Set `worktree.baseRef: "head"` in config to preserve unpushed commits when creating worktrees. Added `$CLAUDE_EFFORT` env var for hooks; fixed credential race conditions causing 401s in parallel sessions. **Why it matters:** Unpushed work will be lost in new worktrees unless you update config.

### [Claude Code v2.1.132: Terminal Mode & Memory Leak Fixes](https://github.com/anthropics/claude-code/releases/tag/v2.1.132)
SIGINT now gracefully restores terminal instead of abrupt exits. Fixed emoji truncation crashes in `--resume`, unbounded memory growth (10GB+ RSS) from MCP stdout pollution, and Windows dead keyboard input. Added `CLAUDE_CODE_DISABLE_ALTERNATE_SCREEN=1` to disable fullscreen rendering for native scrollback. **Why it matters:** Resolves widespread session stability issues and resource exhaustion.

### [Claude Code v2.1.131: Windows Extension Activation Fix](https://github.com/anthropics/claude-code/releases/tag/v2.1.131)
Fixed critical VS Code extension activation failure on Windows caused by hardcoded build paths in bundled SDK (`createRequire` polyfill errors). Restored missing `x-api-key` header in Mantle endpoint requests. **Why it matters:** Unblocks Windows users completely locked out of IDE integration.

## 🆕 New Releases

### [Claude Code v2.1.137: Windows Activation Hotfix](https://github.com/anthropics/claude-code/releases/tag/v2.1.137)
Platform-specific fix for VSCode extension initialization failure on Windows. Incremental patch to v2.1.131's activation bug.

### [Claude Code v2.1.129: Plugin URLs & Output Sync](https://github.com/anthropics/claude-code/releases/tag/v2.1.129)
`--plugin-url <url>` flag enables fetching plugins from URLs for single sessions without local installs. `CLAUDE_CODE_FORCE_SYNC_OUTPUT=1` fixes garbled output in non-standard terminals (Emacs eat). Ctrl+R history picker now searches all projects (pre-2.1.124 behavior); Ctrl+S narrows to current session. `/context` command no longer wastes 1.6k tokens dumping ASCII grids. **Why it matters:** Expands plugin distribution model and fixes edge-case terminal rendering.

### [Claude Code v2.1.128: MCP Reconnect & OTEL Isolation](https://github.com/anthropics/claude-code/releases/tag/v2.1.128)
`--plugin-dir` now accepts `.zip` archives. MCP reconnects no longer flood conversations—tools summarized by server prefix. Critical fix: bash subprocesses no longer inherit CLI's `OTEL_*` env vars (prevented instrumented apps from interfering). `EnterWorktree` creates branches from local HEAD (preserves unpushed commits). `/mcp` command shows tool counts and flags 0-tool servers. **Why it matters:** Improves MCP UX and prevents observability telemetry conflicts.

## 📦 SDK & API Updates

### [Python SDK v0.100.0: Managed Agents API](https://github.com/anthropics/anthropic-sdk-python/releases/tag/v0.100.0)
Adds support for multi-agent orchestration, outcomes, webhooks, and vault validation. Enables building event-driven agent architectures with secure credential management. Fixes webhook configuration issues from v0.99.

### [Python SDK v0.98.0: Auth Overhaul](https://github.com/anthropics/anthropic-sdk-python/releases/tag/v0.98.0)
Workload Identity Federation, interactive OAuth, and auth profiles added—streamlines multi-environment credential management. Headers now configurable via env vars. Fixed `stop_details` propagation in streaming Message objects. **Why it matters:** Simplifies production deployments and reduces hardcoded config.

### [TypeScript SDK v0.95.0: Managed Agents Support](https://github.com/anthropics/anthropic-sdk-typescript/releases/tag/sdk-v0.95.0)
Multi-agent, outcomes, webhooks, and vault validation now available in TypeScript. Enables Node.js developers to build agent systems with Claude's managed platform features.

### [TypeScript SDK v0.93.0: Enterprise Auth](https://github.com/anthropics/anthropic-sdk-typescript/releases/tag/sdk-v0.93.0)
Workload Identity Federation, interactive OAuth, and auth profiles added—credential-less cloud deployments without manual token management.

### [Python Agent SDK v0.1.74: Hook Events & Deferred Tool Use](https://github.com/anthropics/claude-agent-sdk-python/releases/tag/v0.1.74)
Hook event streaming (`include_hook_events`) emits PreToolUse/PostToolUse/Stop as `HookEventMessage`. "Defer" hook decisions via `DeferredToolUse` for deferred round trips. `strict_mcp_config=True` locks CLI to explicitly passed servers (deterministic reproducibility). Post-tool hook output replacement works for all tools. **Why it matters:** Achieves feature parity with TypeScript SDK for advanced agentic workflows.

### [Python Agent SDK v0.1.73: Eager Session Store Flush](https://github.com/anthropics/claude-agent-sdk-python/releases/tag/v0.1.73)
`session_store_flush="eager"` option enables real-time transcript delivery to SessionStore—supports live-tailing UIs, cross-process resume, and crash durability. **Why it matters:** Unblocks live monitoring and recovery workflows.

### [TypeScript Agent SDK v0.2.136: Task Tools Deprecation](https://github.com/anthropics/claude-agent-sdk-typescript/releases/tag/v0.2.136)
**Breaking:** `TodoWrite` deprecated in favor of Task tools (`TaskCreate`, `TaskGet`, `TaskUpdate`, `TaskList`). New `resolveSettings()` alpha method inspects effective merged settings without spawning CLI, reads MDM configs for environment parity.

### [TypeScript Agent SDK v0.2.133: V2 Session API Deprecation](https://github.com/anthropics/claude-agent-sdk-typescript/releases/tag/v0.2.133)
Unstable V2 session API (`unstable_v2_createSession`, `unstable_v2_resumeSession`, `unstable_v2_prompt`) deprecated—migrate to `query()` method. Passing `'Skill'` in `allowedTools` deprecated—use `skills` option.

### [MCP Python SDK v1.27.1: Compatibility Patches](https://github.com/modelcontextprotocol/python-sdk/releases/tag/v1.27.1)
Pydantic 2.13 compatibility fix (catches PydanticUserError in output schemas). OAuth coerces empty-string optional URL fields to None. httpx constrained to <1.0.0 to avoid breaking changes. **Why it matters:** Prevents runtime failures with Pydantic 2.13+.

### [Python Agent SDK v0.1.77: Better Error Messages](https://github.com/anthropics/claude-agent-sdk-python/releases/tag/v0.1.77)
Replaced generic 'Command failed with exit code 1' with actual error text (e.g., 'Reached maximum number of turns'). Deprecated `allowed_tools` string parameter—use `skills` option. Bundled CLI updated to v2.1.133.

### [Python Agent SDK v0.1.76: API Error Status Exposure](https://github.com/anthropics/claude-agent-sdk-python/releases/tag/v0.1.76)
`api_error_status` field in ResultMessage enables safe logging/classification of API failures (429, 500, 529). Fixed ToolPermissionContext.suggestions deserialization to return PermissionUpdate instances instead of dicts.

## 📄 Docs Changes

### [Claude Code Hooks Guide Updated](https://code.claude.com/docs/en/hooks-guide)
Documentation now covers deterministic automation via shell commands at lifecycle events (PostToolUse, Notification, SessionStart, ConfigChange, CwdChanged). Includes common patterns: auto-formatting with Prettier, blocking .env/.git edits via exit code 2, re-injecting context after compaction, auto-approving permission prompts. Verify with `/hooks` CLI command. **Why it matters:** Unlocks workflow automation for compliance, formatting, and context management.

### [MCP Documentation: CLI Workflow Clarifications](https://code.claude.com/docs/en/mcp)
Critical CLI details added: (1) `claude mcp add` options must precede server name with `--` separator. (2) Project-scoped `.mcp.json` shareable in version control. (3) HTTP/SSE servers auto-reconnect (exponential backoff, 5 attempts). (4) Plugin-bundled MCP servers auto-enable with `/reload-plugins`. **Why it matters:** Reduces MCP setup friction and clarifies team collaboration workflows.
