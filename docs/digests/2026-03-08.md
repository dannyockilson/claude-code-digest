## 🚨 Breaking Changes & Action Required

### [Claude Code v2.1.70: Third-party Gateway Support Fixed](https://github.com/anthropics/claude-code/releases/tag/v2.1.70)
`tool_reference` blocks now correctly disabled when using `ANTHROPIC_BASE_URL` with proxies, resolving API 400 errors for users routing through alternative endpoints. Custom Bedrock profiles and non-standard model identifiers now work without `effort parameter` errors. **Why it matters:** If you're using Claude Code behind corporate proxies or with alternative API gateways, this unblocks you from the past month of instability.

### [Claude Code v2.1.69: Nested Skill Security Fix](https://github.com/anthropics/claude-code/releases/tag/v2.1.69)
Nested skill discovery now excludes gitignored directories (security fix for sandbox escapes). Model aliases `claude-opus-4-0/4-1` now resolve correctly to current versions instead of deprecated models. Voice STT expanded to 20 languages total. **Why it matters:** If you have nested skills in version-controlled repos, verify they're not in `.gitignore` or they won't load anymore.

### [Claude Code v2.1.68: Opus 4.6 Effort Defaults Changed](https://github.com/anthropics/claude-code/releases/tag/v2.1.68)
Opus 4.6 now defaults to medium effort for Max/Team subscribers (balancing speed vs. thoroughness); use `ultrathink` keyword to force high effort. Opus 4/4.1 deprecated on first-party API with automatic migration to 4.6. **Why it matters:** Your prompts may run faster but less thoroughly—add `ultrathink` to critical requests if you need deep reasoning.

### [claude-code-action v1.0.65-66: Report Visibility Locked Down](https://github.com/anthropics/claude-code-action/releases/tag/v1.0.65)
`display_report` now defaults to `false`, hiding sensitive CI/CD output by default. v1.0.66 further restricts sanitized outputs to only `permission_denials` count. **Why it matters:** If your workflows relied on full report visibility, explicitly set `display_report: true` or update log parsing to handle reduced metadata.

## 🆕 New Releases

### [Claude Code v2.1.71: Automation & Reliability](https://github.com/anthropics/claude-code/releases/tag/v2.1.71)
Added `/loop` command for recurring prompts at intervals (e.g., `/loop 5m check the deploy`) and cron scheduling tools for session-level automation. Fixed critical stdin freeze in long-running sessions and 5-8s startup delay with voice mode. Voice activation key now rebindable via `voice:pushToTalk` in keybindings.json. Bash auto-approval allowlist expanded with 10+ utilities (`fmt`, `comm`, `cmp`, etc.). **Why it matters:** You can now schedule Claude Code to autonomously monitor deploys or run health checks without leaving a session open.

### [Claude Code v2.1.70: Native MCP Management](https://github.com/anthropics/claude-code/releases/tag/v2.1.70)
New `/mcp` command provides GUI-based server enable/disable and OAuth auth directly in chat—no terminal switching needed. Fixed Windows voice mode, SSH Enter key handling, and clipboard corruption with non-ASCII characters. **Why it matters:** MCP server troubleshooting and toggling is now a first-class chat interaction instead of config file wrangling.

## 📦 SDK & API Updates

### [Python SDK v0.1.48: Streaming Fixed](https://github.com/anthropics/claude-agent-sdk-python/releases/tag/v0.1.48)
Critical regression fixed: `include_partial_messages=True` now correctly delivers `input_json_delta` events (broken in v0.1.36-0.1.47). Bundled Claude CLI updated to 2.1.71. **Why it matters:** If you're streaming tool inputs for real-time UI updates, upgrade immediately—this was silently broken for 12+ versions.

### [Python SDK v0.1.46: Dynamic Server Management](https://github.com/anthropics/claude-agent-sdk-python/releases/tag/v0.1.46)
Added `list_sessions()`, `get_session_messages()`, `add_mcp_server()`, `remove_mcp_server()` for runtime MCP configuration without restarts. Type-safe task messages (`TaskStarted`, `TaskProgress`) and `ResultMessage.stop_reason` improve event handling. Fixed string prompts breaking MCP registration by closing stdin prematurely. **Why it matters:** Agents can now dynamically load/unload MCP capabilities mid-session, enabling adaptive tool discovery workflows.

### [TypeScript SDK v0.2.70: HTTP MCP Servers Unblocked](https://github.com/anthropics/claude-agent-sdk-typescript/releases/tag/v0.2.70)
Fixed HTTP MCP servers failing with 406 errors on strict `Accept` header validation. Made `AgentToolInput.subagent_type` optional with `general-purpose` default. Synced with Claude Code v2.1.70. **Why it matters:** If your MCP servers run over HTTP (common for enterprise tool integrations), this fixes a critical blocker.

### [TypeScript SDK v0.2.69: Event Naming Revert](https://github.com/anthropics/claude-agent-sdk-typescript/releases/tag/v0.2.69)
`system:init` and `result` events revert to emitting `'Task'` as Agent tool name (was `'Agent'` in recent patch). Added `toolConfig.askUserQuestion.previewFormat` for `'markdown'` or `'html'` previews, `supportsFastMode` field to `ModelInfo`, and `agent_id`/`agent_type` to hook events. Fixed SDK-mode MCP servers disconnecting during background plugin installation. **Why it matters:** If you parse event streams by tool name, verify your code handles the `'Task'` → `'Agent'` naming churn.

## 📄 Docs Changes

### [Claude Code Hooks Guide: Lifecycle Automation](https://code.claude.com/docs/en/hooks-guide)
Hooks documentation now covers `/hooks` interactive CLI menu for lifecycle event handlers. Key events: `SessionStart`, `PreToolUse` (can block commands), `PostToolUse`, `Notification`, `ConfigChange`, `WorktreeCreate`. Hooks run shell commands, HTTP requests, or LLM-based evaluations with exit code 0 (allow) or 2 (block + feedback). **Why it matters:** You can now auto-format code, send desktop alerts, block protected file edits, or re-inject context at specific lifecycle points without custom shell wrappers.

### [Claude Code MCP Docs: Dynamic Tools & Scopes](https://code.claude.com/docs/en/mcp)
MCP docs expanded with dynamic tool updates via `list_changed` notifications (servers update capabilities without reconnection), plugin-bundled MCP auto-start, and three installation scopes (local overrides project/user). Environment variable expansion in `.mcp.json` (${VAR} and ${VAR:-default}) allows sharing configs with machine-specific secrets. **Why it matters:** Teams can now commit shared MCP configs to git while keeping API keys and local paths flexible per developer.
