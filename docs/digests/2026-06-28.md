## 🚨 Breaking Changes & Action Required

### [Claude Code v2.1.195 - Hook Matcher Pattern Change](https://github.com/anthropics/claude-code/releases/tag/v2.1.195)
Hyphenated hook identifiers (e.g., `mcp__brave-search`) now require exact matching instead of substring matching. If you use hook matchers with MCP server names containing hyphens, update to regex patterns like `mcp__brave-search__.*` for wildcard behavior. **Why it matters:** Existing hooks may silently stop firing without migration.

### [Claude Code v2.1.187 - Sandbox Credential Isolation](https://github.com/anthropics/claude-code/releases/tag/v2.1.187)
New `sandbox.credentials` setting blocks sandboxed commands from accessing credential files and secret environment variables by default. Review your sandbox workflows—if agents need credential access, you must explicitly configure this. Also includes 5-minute MCP tool call timeout (override via `CLAUDE_CODE_MCP_TOOL_IDLE_TIMEOUT`). **Why it matters:** Critical security control for multi-tenant or untrusted code execution scenarios.

### [Claude Code v2.1.186 - Auto-Response to Bash Commands](https://github.com/anthropics/claude-code/releases/tag/v2.1.186)
`!` bash commands now auto-trigger Claude responses by default. Opt out with `respondToBashCommands: false` in settings. Includes new CLI auth commands (`claude mcp login/logout <name>`) for headless MCP server workflows. **Why it matters:** Changes default interaction model—may generate unexpected agent turns if you use bash commands for manual inspection.

### [MCP Python SDK v2.0.0a3 - Protocol v2026-07-28 Stateless Migration](https://github.com/modelcontextprotocol/python-sdk/releases/tag/v2.0.0a3)
Complete protocol overhaul: `initialize` handshake replaced with self-describing POSTs, `ServerRunner` split into three drivers, `ClientSession` gains `.discover()/.adopt()` modes. Wire types moved to standalone `mcp-types` package (import from `mcp_types` not `mcp.types`). Existing code requires migration via new `mode='auto'|'legacy'|<version>` on `Client`. Multi-round tool calls now supported via `InputRequiredResult`. **Why it matters:** No backward compatibility—existing MCP Python integrations will break without code changes.

### [GitHub Actions v1.0 GA - Workflow Syntax Updates](https://code.claude.com/docs/en/github-actions)
Action upgraded from beta with breaking changes: rename `direct_prompt` → `prompt`, remove `mode` config, move `max_turns`/`model`/`custom_instructions` to unified `claude_args` parameter. New capability: invoke skills directly and use Amazon Bedrock/Google Vertex AI for data residency. **Why it matters:** Beta workflows will fail after upgrade without syntax migration.

## 🆕 New Releases

### [Claude Code v2.1.193 - Strict Shell Execution Control](https://github.com/anthropics/claude-code/releases/tag/v2.1.193)
Added `autoMode.classifyAllShell` setting to route all Bash/PowerShell commands through auto-mode classifier, enabling uniform security policy enforcement. Includes live file path autocomplete in bash mode, automatic memory-pressure reaping for idle shell commands, and OpenTelemetry assistant response logging (opt-in via `OTEL_LOG_ASSISTANT_RESPONSES`). Fixed critical background agent bugs: spurious cancellations, phantom subagent spawning, and pinned agents re-prompting after auto-updates. **Why it matters:** First unified security control point for shell operations across all execution paths.

### [Claude Code v2.1.191 - Conversation Recovery](https://github.com/anthropics/claude-code/releases/tag/v2.1.191)
`/rewind` command now allows resuming conversations after `/clear`—major workflow restoration for accidental clears. Background agents now resurrect instead of permanently stopping after errors, with improved MCP retry logic for transient failures and OAuth handling. Streaming CPU usage reduced ~37% via 100ms text update coalescing. Fixed 10+ UI/UX bugs including Windows Terminal URL truncation, Ghostty link clicks, comma-separated hook matchers, and long-session memory leaks.

## 🔌 MCP Ecosystem

### [Graphenium - MCP-Native Codebase Graph Tool](https://github.com/lambda-alpha-labs/Graphenium)
Rust-based MCP server that builds queryable graph representations of codebases, replacing grep-based context gathering with structured graph queries. Trust-aware design enables efficient large-repo navigation without loading entire files. **Why it matters:** First production-ready tool addressing the core context efficiency problem for AI agents on complex codebases.

### [MCP TypeScript SDK - Legacy v1 Migration Package](https://github.com/modelcontextprotocol/typescript-sdk/releases/tag/%40modelcontextprotocol/server-legacy%402.0.0-alpha.3)
New `@modelcontextprotocol/server-legacy@2.0.0-alpha.3` package provides frozen v1 SSE transport and OAuth Authorization Server helpers. **Why it matters:** Enables gradual v1→v2 migration without breaking production deployments.

### [MCP HTTP/SSE Auto-Reconnection](https://code.claude.com/docs/en/mcp)
HTTP/SSE MCP servers now support automatic reconnection with exponential backoff (up to 5 attempts) and capability discovery retries on transient errors. Dynamic tool updates via `MCPlist_changed` notifications allow servers to refresh available tools/prompts/resources without disconnect/reconnect cycles. WebSocket servers gain header-based authentication with static tokens or `headersHelper` for runtime generation. **Why it matters:** Significantly reduces manual intervention for flaky remote MCP connections in production workflows.
