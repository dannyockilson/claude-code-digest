## 🚨 Breaking Changes & Action Required

### **Claude Code GitHub Actions v1.0 GA**
[View Documentation](https://code.claude.com/docs/en/github-actions)

Breaking changes from beta require workflow updates:
- **Action**: Update `@beta` → `@v1` in workflow YAML
- **Rename**: `direct_prompt` → `prompt`, remove `mode` parameter (now auto-detected)
- **Consolidate**: Move `max_turns`, `model`, `custom_instructions` into unified `claude_args` flag structure
- New enterprise support for AWS Bedrock and Google Vertex AI with Workload Identity Federation for data residency compliance.

**Why it matters**: Auto-detection simplifies configuration but breaks existing beta workflows—audit CI pipelines before Monday.

---

### **Claude Code v2.1.49: Ctrl+C behavior change**
[Release Notes](https://github.com/anthropics/claude-code/releases/tag/v2.1.49)

Ctrl+C/ESC now requires **two presses within 3 seconds** to kill background agents (prevents accidental termination). Sonnet 4.5 1M context **deprecated on Max plan** in favor of Sonnet 4.6 1M. Simple mode now includes file edit tool alongside Bash.

**Why it matters**: If your workflows rely on single-press interrupts or Sonnet 4.5's extended context, update scripts and model references immediately.

---

### **Claude Agent SDK TypeScript v0.2.45: Stream behavior fix**
[Release Notes](https://github.com/anthropics/claude-agent-sdk-typescript/releases/tag/v0.2.45)

`Session.stream()` now correctly waits for background subagents to complete before returning—previous versions returned prematurely. If you depend on early stream termination, refactor to handle new blocking behavior.

### **Claude Code v2.1.50: Worktree isolation + memory leak fixes**
[Release Notes](https://github.com/anthropics/claude-code/releases/tag/v2.1.50)

- **Worktree isolation**: Declarative `isolation: worktree` in agent definitions with `WorktreeCreate`/`WorktreeRemove` hooks enables VCS-sandboxed multi-agent workflows
- **Memory leak resolution**: Fixed 7+ leaks (LSP diagnostics, task output, CircularBuffer, MCP tools, completed tasks) causing unbounded growth in long sessions
- **Performance**: Headless mode (-p) defers WASM/UI imports; new `claude agents` CLI command
- **Simple mode hardening**: Strips MCP tools, attachments, hooks, CLAUDE.md, and skills for minimal environments

**Why it matters**: Critical stability update for production deployments—worktree isolation prevents agent cross-contamination; memory fixes enable multi-hour sessions without restarts.

### **Claude Code MCP: Dynamic tool updates and scope hierarchy**
[View Documentation](https://code.claude.com/docs/en/mcp)

- **Dynamic tool updates**: `list_changed` notifications enable live capability refresh without reconnecting
- **Scope hierarchy**: local (project-private ~/.claude.json), project (team-shared .mcp.json in VCS), user (cross-project ~/.claude.json)
- **Plugin bundling**: Plugins auto-bundle MCP servers via .mcp.json or plugin.json—tools appear automatically
- **Environment variables**: `${VAR}` and `${VAR:-default}` syntax in .mcp.json enables secure credential sharing

**Why it matters**: Scope hierarchy critical for team collaboration—project-level .mcp.json in version control standardizes tooling. Dynamic updates eliminate reconnection overhead for evolving integrations.
