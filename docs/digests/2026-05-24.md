## 🚨 Breaking Changes & Action Required

### Claude Code v2.1.148: Critical Bash Exit Code Regression
**[Release Notes](https://github.com/anthropics/claude-code/releases/tag/v2.1.148)**

One-version regression in v2.1.147 caused all Bash commands to incorrectly return exit code 127 instead of actual exit codes. **If you're on v2.1.147, upgrade immediately**—this breaks command validation, CI/CD workflows, and any automation relying on exit codes.

**Why it matters:** Exit code 127 typically signals "command not found," so automated scripts likely misinterpreted successful operations as failures.

---

### Claude Code v2.1.147: `/simplify` Renamed to `/code-review`
**[Release Notes](https://github.com/anthropics/claude-code/releases/tag/v2.1.147)**

The `/simplify` command is now `/code-review` with new syntax: effort levels (`--low`, `--medium`, `--high`) and a `--comment` flag for GitHub PR inline comments. Old cleanup-and-fix behavior removed. Pinned background sessions (`Ctrl+T`) now persist when idle, auto-restart for updates, and survive memory pressure shedding. Fixed 25+ bugs across PowerShell, MCP pagination, auto mode, and Windows Terminal rendering.

**Why it matters:** Muscle memory update required for power users; new PR integration enables tighter GitHub workflow coupling.

---

### Claude Code v2.1.146: MCP Pagination Bug Fixed
**[Release Notes](https://github.com/anthropics/claude-code/releases/tag/v2.1.146)**

Fixed critical bug where MCP resource/template/prompt lists dropped all items past page 1. Also renamed `/simplify` → `/code-review` (breaking for muscle memory), fixed Windows PowerShell regression (v2.1.124) where `pwsh` via winget/Store crashed with "invalid command line" errors, and resolved Windows Terminal strobing during streaming.

**Why it matters:** Large model context providers were silently truncating available resources; workflows depending on full resource lists were incomplete.

---

### Claude Code v2.1.144: `/model` Command Behavior Change
**[Release Notes](https://github.com/anthropics/claude-code/releases/tag/v2.1.144)**

`/model` now affects current session only; press `d` in picker to set default for new sessions. Critical startup fix: API calls timeout at 15s instead of 75s hang when `api.anthropic.com` is unreachable (captive portal/firewall/VPN). `/resume` now supports `claude --bg` background sessions. 30+ bug fixes across terminal rendering, MCP servers, and remote auth.

**Why it matters:** Multi-session workflows need new mental model for per-session vs. default model selection; 15s timeout prevents frozen startups in network-constrained environments.

## 🆕 New Releases

### Claude Code v2.1.149: Granular Usage Tracking
**[Release Notes](https://github.com/anthropics/claude-code/releases/tag/v2.1.149)**

`/usage` now provides per-category breakdown (skills, subagents, plugins, MCP servers) for limit tracking across complex workflows. Fixed critical PowerShell permission bypass where built-in `cd` functions changed directories undetected, improved sandbox git-worktree allowlist, and added keyboard navigation (arrows, j/k, PgUp/PgDn, Space, Home/End) to `/diff` detail view for hands-free code review.

**Why it matters:** Granular usage data enables precise consumption management when orchestrating multiple agent types; permission bypass fix closes sandbox escape vector.

---

### Claude Code v2.1.145: JSON Output for Automation
**[Release Notes](https://github.com/anthropics/claude-code/releases/tag/v2.1.145)**

Added `claude agents --json` for scripting and session monitoring (tmux, status bars, session pickers). Fixed critical permission bypass where bare variable assignments to non-allowlisted env vars in Bash were auto-approved. Improved OTEL tracing with `agent_id`/`parent_agent_id` spans for subagent debugging. 15+ bug fixes including voice push-to-talk, task list ordering, and infinite loop in fork-context skills.

**Why it matters:** JSON output unblocks automated agent lifecycle management; env var bypass fix closes security hole in multi-agent workflows.
