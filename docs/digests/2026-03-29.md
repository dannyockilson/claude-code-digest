## 🚨 Breaking Changes & Action Required

**[Claude Code v2.1.86](https://github.com/anthropics/claude-code/releases/tag/v2.1.86)** — Critical regression fixes for session continuity  
Fixed `--resume` failing on pre-v2.1.85 sessions and `--bare` mode dropping MCP tools—both breaking workflows that depend on session restoration. Also fixed Write/Edit/Read operations failing on files outside project root when rules are configured. Reduced token overhead by eliminating JSON escaping of raw file content with `@` mentions and deduplicating line numbers in read operations. Fixed macOS/Linux plugin installation permissions and config corruption from excessive disk writes.  
**Why it matters:** If you're running pre-v2.1.86, session resume is broken and file access may be unexpectedly restricted—upgrade immediately.

**[Claude Code v2.1.83](https://github.com/anthropics/claude-code/releases/tag/v2.1.83)** — Managed settings now support drop-in directory  
Added `managed-settings.d/` for team-based policy deployment—policy fragments merge alphabetically alongside monolithic `managed-settings.json`. New `CLAUDE_CODE_SUBPROCESS_ENV_SCRUB=1` strips credentials from subprocess environments (Anthropic tokens, cloud provider creds, GitHub Actions OIDC). 30+ UI/UX bug fixes including startup hangs (3s MCP fetch, 1-8s audio module) and macOS hang-on-exit.  
**Why it matters:** Multi-team environments can now deploy independent policy fragments without merge conflicts; subprocess credential scrubbing is critical for sandboxed operations.

**[claude-code-action v1.0.77](https://github.com/anthropics/claude-code-action/releases/tag/v1.0.77)** — Auto-enables subprocess credential scrubbing  
Auto-enables `CLAUDE_CODE_SUBPROCESS_ENV_SCRUB=1` for workflows using `allowed_non_write_users` to block prompt injection attacks that could leak API keys via shell expansion. Scrubs Anthropic tokens, cloud provider credentials, GitHub Actions OIDC/runtime tokens, OTEL auth—but preserves `GITHUB_TOKEN` for GitHub API calls. Requires Claude Code v2.1.79+.  
**Why it matters:** Prevents credential leaks in CI/CD workflows; opt-out via `CLAUDE_CODE_SUBPROCESS_ENV_SCRUB: "0"` if subprocesses legitimately need inherited credentials.

## 🆕 New Releases

**[Claude Code v2.1.85](https://github.com/anthropics/claude-code/releases/tag/v2.1.85)** — Multi-server MCP support and conditional hooks  
MCP environment variables now support multi-server configurations with helper scripts. Conditional hooks with permission syntax (e.g., `Bash(git *)`) reduce process spawning overhead. Fixed critical `/compact` failure on large conversations and MCP step-up authorization flow with refresh tokens. Terminal keyboard protocol fix resolves Ctrl+C/D handling in Ghostty, Kitty, WezTerm.  
**Why it matters:** Multi-server MCP setups are now officially supported; conditional hooks reduce overhead for permission-aware automation.

**[Claude Code v2.1.84](https://github.com/anthropics/claude-code/releases/tag/v2.1.84)** — PowerShell preview and 3P model detection  
PowerShell tool now available as opt-in preview for Windows users. New env vars for model capability detection (`ANTHROPIC_DEFAULT_{OPUS,SONNET,HAIKU}_MODEL_SUPPORTS`) and streaming timeout configuration (`CLAUDE_STREAM_IDLE_TIMEOUT_MS`)—critical for Bedrock, Vertex, Foundry integrations. CLI/UX fixes: `Ctrl+U` works on multiline input, arrow keys responsive in footers, deep links open in preferred terminal, ~30ms startup perf improvement. MCP deduplication (local config wins) and tool description capping at 2KB reduce context bloat.  
**Why it matters:** Windows users get native PowerShell support; 3P model integrations can now properly detect capabilities and configure timeouts.

## 📦 SDK & API Updates

**[claude-agent-sdk-python v0.1.52](https://github.com/anthropics/claude-agent-sdk-python/releases/tag/v0.1.52)** — Context introspection and tool metadata improvements  
Breaking fix: `connect(prompt="...")` now works correctly (previously silently dropped string prompts causing `receive_messages()` hangs). New `get_context_usage()` method exposes per-category context window consumption for token budgeting. `@tool` decorator and `create_sdk_mcp_server` now support `typing.Annotated` for per-parameter descriptions. Exposed `tool_use_id` and `agent_id` in `ToolPermissionContext` for tracking parallel permission requests.  
**Why it matters:** Context introspection is critical for token-aware agent design; the prompt fix unblocks basic SDK usage.

**[claude-agent-sdk-python v0.1.51](https://github.com/anthropics/claude-agent-sdk-python/releases/tag/v0.1.51)** — Session management overhaul  
Session management overhaul with `fork_session()`, `delete_session()`, and pagination support. New `task_budget` option for token limits and `--system-prompt-file` CLI flag. Added `disallowedTools`, `maxTurns`, `initialPrompt` to AgentDefinition. 13 bug fixes including Python 3.10 TypedDict compatibility, async generator cleanup, MCP tool input schema conversion.  
**Why it matters:** Session forking enables agent state branching for exploration/rollback workflows; task budgets enforce token discipline.

**[claude-agent-sdk-typescript v0.2.86](https://github.com/anthropics/claude-agent-sdk-typescript/releases/tag/v0.2.86)** — Context usage and session auto-assignment  
Added `getContextUsage()` method for context window breakdown by category. Made `session_id` optional in `SDKUserMessage` (SDK auto-assigns). Fixed TypeScript type resolution by adding `@anthropic-ai/sdk` and `@modelcontextprotocol/sdk` as explicit dependencies—improves IDE/LSP accuracy. Parity with Claude Code v2.1.86.

**[claude-agent-sdk-typescript v0.2.85](https://github.com/anthropics/claude-agent-sdk-typescript/releases/tag/v0.2.85)** — Runtime plugin reload  
Added `reloadPlugins()` method for runtime plugin/MCP server refresh without restart. Fixed PreToolUse permission hooks with `permissionDecision: "ask"` being silently ignored—now correctly prompts for user approval. Synced to Claude Code v2.1.85 parity.  
**Why it matters:** Runtime plugin reload enables dynamic tool ecosystems without restarting agent processes.

**[claude-agent-sdk-typescript v0.2.84](https://github.com/anthropics/claude-agent-sdk-typescript/releases/tag/v0.2.84)** — Task budget and channel activation  
Added `taskBudget` option for token-aware tool pacing. New `enableChannel()` method and `capabilities` field on `McpServerStatus` enable SDK-driven MCP channel activation. Exported `EffortLevel` type. Fixed false-positive "[Request interrupted by user]" error messages.

**[claude-agent-sdk-typescript v0.2.83](https://github.com/anthropics/claude-agent-sdk-typescript/releases/tag/v0.2.83)** — Persistent read state  
Added `seed_read_state` control subtype to persist file read state ({path, mtime}) enabling `Edit` operations after originating `Read` removed from context—solves context window constraints in long-running agents. Made `session_state_changed` events opt-in via `CLAUDE_CODE_EMIT_SESSION_STATE_EVENTS=1`.  
**Why it matters:** Persistent read state enables edits in long-running agents without keeping original reads in context—critical for token efficiency.

**[MCP TypeScript SDK v1.28.0](https://github.com/modelcontextprotocol/typescript-sdk/releases/tag/v1.28.0)** — OAuth scope defaults and memory leak fixes  
Resource metadata now uses `scopes_supported` by default, improving OAuth scope handling. Server auth defaults to `client_secret_basic` when `token_endpoint_auth_methods_supported` is omitted (RFC 8252 compliance). Fixed rejection of plain JSON Schema objects passed as inputSchema. Fixed memory leaks in connection cleanup: `_timeoutInfo` now properly cleared in `_onclose()` and abort controller scope finalized correctly.

## 📄 Docs Changes

**[Claude Code Overview](https://code.claude.com/docs/en/overview)** — Feature map and recommended entry points  
Comprehensive feature map added across all surfaces (Terminal CLI, VS Code, Desktop, Web, JetBrains). Terminal CLI is now the primary recommended entry point with native auto-updating installs via Homebrew/WinGet (Git for Windows required). MCP integration, CLAUDE.md project instructions, and agent team orchestration now prominently featured. Cross-surface session continuity expanded: `/teleport` moves terminal work to web, `/desktop` hands off to visual diff review, Remote Control enables phone continuation.

**[Claude Code Hooks Guide](https://code.claude.com/docs/en/hooks-guide)** — Lifecycle automation triggers  
New documentation for lifecycle hooks—automation triggers at key points (session start, tool use, permission requests, file changes). Key use cases: desktop notifications when Claude awaits input, auto-format code post-edit with Prettier, block edits to sensitive files (.env, .git/), re-inject context after compaction, audit config changes, reload environment on directory changes (direnv integration), auto-approve specific permission prompts. Hooks use `~/.claude/settings.json` with event matchers; matching hooks execute in parallel with deduplication.  
**Why it matters:** Hooks enable reactive automation without polling—critical for integrating external tools (formatters, linters, monitors) into Claude workflows.

**[Claude Code MCP](https://code.claude.com/docs/en/mcp)** — Dynamic tools and plugin-bundled servers  
Major MCP documentation overhaul: (1) Dynamic tool updates via `list_changed` notifications—servers can update capabilities without reconnection, enabling real-time tool availability changes; (2) Plugin-bundled MCP servers with automatic lifecycle management (`.mcp.json` support, `/reload-plugins` command) eliminating manual setup; (3) Three-tier scope hierarchy (local/project/user) with precedence rules for conflict resolution; (4) Environment variable expansion in `.mcp.json` (`${VAR}` syntax) for team-shareable configs with machine-specific flexibility.  
**Why it matters:** Dynamic tool updates and plugin-bundled servers reduce friction for MCP adoption; env var expansion solves team config sharing.
