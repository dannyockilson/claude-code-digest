## 🚨 Breaking Changes & Action Required

### [claude-code v2.1.154](https://github.com/anthropics/claude-code/releases/tag/v2.1.154)
**Opus 4.8 now default; workflow behavior changes**  
Opus 4.8 becomes the default model with high-effort mode that orchestrates 10s-100s of background agents for complex tasks. Fast mode costs 2x standard rate for 2.5x speed. `/simplify` now does cleanup-only review instead of full bug-hunting. Lean system prompt becomes default; `/effort` labels renamed to 'Faster'/'Smarter'. Background shell execution via `! <command>` or `claude --bg --exec`.  
**Why it matters:** Existing workflows may behave differently; review effort settings and task delegation patterns.

### [claude-code v2.1.147](https://github.com/anthropics/claude-code/releases/tag/v2.1.147)
**`/simplify` renamed to `/code-review` with new syntax**  
Breaking: `/simplify` renamed to `/code-review` with effort-level syntax (e.g., `/code-review high`) and `--comment` flag for inline GitHub PR comments. Cleanup-fix behavior removed. Also includes 30+ fixes for PowerShell corruption, Windows Terminal strobing, MCP pagination data loss, and enterprise login enforcement.  
**Why it matters:** Update all scripts and workflows referencing `/simplify`; review code review automation patterns.

### [claude-code v2.1.157](https://github.com/anthropics/claude-code/releases/tag/v2.1.157)
**Critical sandbox permission fix + auto-load plugins**  
Fixed critical sandbox permission prompt regression in auto/bypass modes across desktop/IDE/SDK that caused unexpected automation interruptions. Plugins now auto-load from `.claude/skills` directories without marketplace requirement; new `claude plugin init` scaffolding command. Enhanced `claude agents` session management: `agent` field in `settings.json` honored, `EnterWorktree` switches worktrees mid-session.  
**Why it matters:** Eliminates unexpected prompts breaking automation; reduces plugin setup friction.

### [GitHub Actions v1.0](https://code.claude.com/docs/en/github-actions)
**GA release with breaking changes from beta**  
Rename `direct_prompt` → `prompt`, remove `mode` config (auto-detected), consolidate CLI options into `claude_args` parameter, update action reference from `@beta` to `@v1`. Claude Opus 4.8 now available via `claude-opus-4-8` model parameter. New enterprise support for Bedrock/Vertex with Workload Identity Federation.  
**Why it matters:** All beta workflows must migrate to v1 syntax; review authentication patterns for enterprise deployments.

### [claude-code v2.1.148](https://github.com/anthropics/claude-code/releases/tag/v2.1.148)
**Critical Bash tool regression fix**  
Fixes critical regression from v2.1.147 causing exit code 127 (command not found) on every Bash command for some users, completely blocking CLI workflow execution.  
**Why it matters:** Update immediately if stuck on v2.1.147; restores basic command execution reliability.

### [claude-code-action v1.0.133](https://github.com/anthropics/claude-code-action/releases/tag/v1.0.133)
**Workload identity federation support**  
Now supports OIDC workload identity federation for Claude auth in CI workflows—eliminates need to store API keys as GitHub secrets, using short-lived tokens instead.  
**Why it matters:** Improves security posture by removing static credentials from CI/CD pipelines.

## 📄 Docs Changes

### [Claude Code Hooks Guide](https://code.claude.com/docs/en/hooks-guide)
**Hooks now primary extension mechanism**  
Comprehensive guide documents deterministic lifecycle control without LLM judgment. 6 ready-to-use configs: desktop notifications, auto-formatting (Prettier), protected file blocking, context re-injection after compaction, audit logging, environment reloading (direnv/devbox). Matcher system for fine-grained event filtering (permission_prompt, idle_prompt, auth_success, elicitation_* for MCP servers). Hooks in settings.json verified via `/hooks` CLI; block actions with exit code 2.  
**Why it matters:** Replaces manual workflow integration with systematic automation; deterministic control > prompt engineering.

### [Claude Code MCP](https://code.claude.com/docs/en/mcp)
**Expanded MCP integration patterns**  
Documents HTTP/SSE/stdio/WebSocket server options with explicit `claude mcp add --transport stdio myserver -- npx server` syntax. Dynamic tool updates via `list_changed` notifications, automatic reconnection with exponential backoff. New `CLAUDE_PROJECT_DIR` environment variable for stdio servers enables project-relative paths without working directory dependencies.
