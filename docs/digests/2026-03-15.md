## 🚨 Breaking Changes & Action Required

### [Claude Code GitHub Actions v1.0 GA](https://code.claude.com/docs/en/github-actions)
Major workflow config changes in production release: `mode` parameter removed (auto-detected), `direct_prompt` renamed to `prompt`, CLI options consolidated under `claude_args`. Model default upgraded to Claude Sonnet; Opus 4.6 now available via `claude-opus-4-6` parameter. Added AWS Bedrock & Google Vertex AI support for enterprise data residency. **Action required:** Update workflow YAML files—beta config syntax no longer supported. Consider adding `--max-turns` limits to manage CI token costs.

### [Claude Code v2.1.73 `/output-style` Deprecation](https://github.com/anthropics/claude-code/releases/tag/v2.1.73)
`/output-style` command deprecated in favor of `/config`; output style now locked at session start for prompt caching optimization. **Why it matters:** Runtime style switching breaks cache effectiveness—fix your workflows to set style via config files instead of mid-session commands.

## 🆕 New Releases

### [Claude Code v2.1.76](https://github.com/anthropics/claude-code/releases/tag/v2.1.76)
MCP elicitation support enables interactive form/browser dialogs during tasks—critical for multi-step workflows requiring mid-task input. New `Elicitation` hooks let you intercept/override responses. CLI gains `-n` flag for session naming and `/effort` command for model effort control. Major fixes: deferred tools losing schemas after compaction, plan mode re-approval loops, voice mode on Windows, Remote Control session recovery after WebSocket disconnects. **Why it matters:** First-class support for interactive MCP servers removes manual polling workarounds.

### [Claude Code v2.1.74](https://github.com/anthropics/claude-code/releases/tag/v2.1.74)
Memory leak fix for streaming API buffers prevents unbounded RSS growth on Node.js/npm workflows—critical for long-running sessions. `/context` command now provides actionable optimization suggestions (context-heavy tools, memory bloat, capacity warnings) instead of raw metrics. Fixed model ID handling: full IDs like `claude-opus-4-5` now work in agent frontmatter and `--agents` JSON config. Multiple MCP OAuth fixes (callback port conflicts, refresh token expiry re-auth) and security fix (managed policy `ask` rules no longer bypassed). **Why it matters:** Memory leak caused session crashes in 4+ hour workflows; `/context` optimization hints reduce token waste.

### [Claude Code v2.1.73](https://github.com/anthropics/claude-code/releases/tag/v2.1.73)
Critical fix: Bash tool output no longer lost when running multiple Claude Code sessions in same directory—major reliability issue for concurrent workflows. Fixed deadlock freezing Claude Code when many skill files change simultaneously (e.g., `git pull` in large `.claude/skills/` dirs) and permission prompt CPU loops. Added `modelOverrides` setting for custom provider model IDs (Bedrock ARNs, Vertex, Microsoft Foundry) plus improved OAuth/SSL error messaging for corporate proxies. **Why it matters:** Multi-session output corruption was a silent data loss bug affecting team workflows.

### [Claude Code v2.1.72](https://github.com/anthropics/claude-code/releases/tag/v2.1.72)
SSH-friendly `/copy w` command writes focused selection directly to file, bypassing clipboard—major workflow improvement for remote development. Reduced false-positive bash permission prompts via tree-sitter parsing improvements (handles `find -exec`, variable assignments, substitutions)—fewer interruptions. `/plan` now accepts optional description to enter plan mode immediately (e.g., `/plan fix the auth bug`). Expanded tool auto-approval allowlist adds `lsof`, `pgrep`, `tput`, `ss`, `fd`, `fdfind` for read-only operations; improved prompt cache validation reducing input costs up to 12x. **Why it matters:** Clipboard-free file export unblocks SSH workflows; prompt cache validation cuts repeat costs dramatically.
