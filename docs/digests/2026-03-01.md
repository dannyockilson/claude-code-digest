## 🆕 New Releases

### **claude-code v2.1.63: Memory Leak Fixes + Workflow Commands**
[Release Notes](https://github.com/anthropics/claude-code/releases/tag/v2.1.63)

Critical memory leak fixes for long-running sessions (git cache, JSON parsing, MCP reconnects) plus workflow improvements: new `/simplify` and `/batch` commands, HTTP hooks for custom integrations, project configs now shared across git worktrees, and improved `/model` command showing active model in menu.

Memory leak fixes target unbounded growth in long sessions—particularly git root detection, JSON parsing cache, and MCP server reconnects that were causing observable performance degradation. HTTP hooks enable posting JSON to URLs instead of shell commands, expanding automation options beyond bash. Project config/memory sharing across git worktrees solves multi-branch workflow friction for developers using worktree-based development.

---

### **claude-code v2.1.59: `/copy` Command + Auto-Memory**
[Release Notes](https://github.com/anthropics/claude-code/releases/tag/v2.1.59)

Introduces `/copy` command for selective code block extraction and auto-memory context saving via `/memory`. `/copy` provides interactive picker for individual code blocks or full responses—eliminates manual copying workflow. Auto-memory now captures useful context automatically; manage with `/memory` command—reduces context re-entry friction. Improved bash prefix suggestions for compound commands (e.g., `cd /tmp && git fetch && git push`)—smarter per-subcommand safety prefixes instead of monolithic treatment.

---

### **claude-code v2.1.53: Bulk Agent Kill + Stability Fixes**
[Release Notes](https://github.com/anthropics/claude-code/releases/tag/v2.1.53)

Fixed bulk agent kill (ctrl+f) to send single aggregate notification and properly clear command queue—reduces noise when terminating multiple agents. Resolved UI flicker where user input briefly disappears after submission before message renders. Fixed `--worktree` flag being ignored on first launch. Multiple stability fixes: graceful shutdown stale sessions (Remote Control), Windows panics/crashes, WebAssembly interpreter crashes, and ARM64 timeout crashes.

---

### **claude-code v2.1.51: Remote Control + Security Hardening**
[Release Notes](https://github.com/anthropics/claude-code/releases/tag/v2.1.51)

Added `claude remote-control` subcommand for external builds with local environment serving. BashTool now skips login shell by default when snapshot available (removes need for `CLAUDE_BASH_NO_LOGIN=true`). Tool results >50K chars now persisted to disk (reduced from 100K) to preserve context window. Fixed security issues: workspace trust validation for hooks, HTTP env var interpolation requires explicit allowlist, and duplicate control_response handling from WebSocket reconnects.

**Why it matters**: Security fixes prevent unauthorized execution of hooks and env var injection attacks in team environments.

## 📄 Docs Changes

### **Claude Code Hooks Guide: Expanded Walkthrough**
[Documentation](https://code.claude.com/docs/en/hooks-guide)

Documentation expanded with practical walkthrough: `/hooks` interactive CLI menu now primary entry point for creating hooks (desktop notifications, auto-formatting, file blocking). New hook types beyond shell commands: HTTP POST, prompt-based LLM evaluation, agent-based multi-turn verification. Key lifecycle events documented with timing details (SessionStart, PreToolUse, PostToolUse, ConfigChange, WorktreeCreate, etc.). Hooks run in parallel with automatic deduplication; control flow via exit codes (0=allow, 2=block) and stdout/stderr for context injection or feedback.

**Why it matters**: Hooks are the primary automation mechanism for Claude Code—this guide significantly lowers the barrier to custom workflow integration.

---

### **Claude Code MCP: Dynamic Updates + Plugin-Bundled Servers**
[Documentation](https://code.claude.com/docs/en/mcp)

Dynamic tool updates via MCPlist_changed notifications—MCP servers can now refresh available tools/prompts/resources without requiring disconnect/reconnect, improving workflow continuity. Plugin-bundled MCP servers now supported—plugins can define MCPs in .mcp.json or plugin.json that auto-start when enabled, eliminating manual MCP configuration for teams. Three MCP installation scopes (local/project/user) with clear precedence hierarchy—local scope overrides project/user, enabling personal configs alongside team-shared servers checked into version control. Enhanced MCP authentication—OAuth 2.0 support with pre-configured credentials for servers lacking dynamic client registration.

**Why it matters**: Plugin-bundled MCPs dramatically reduce configuration overhead for teams distributing custom tooling.
