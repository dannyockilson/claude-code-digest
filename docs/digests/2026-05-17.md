## 🚨 Breaking Changes & Action Required

### **Claude Code v2.1.142: Opus 4.7 Default + Agent CLI Overhaul**
[Release Notes](https://github.com/anthropics/claude-code/releases/tag/v2.1.142)

Fast mode now defaults to Opus 4.7 instead of 4.6. Set `CLAUDE_CODE_OPUS_4_6_FAST_MODE_OVERRIDE=1` to revert if you need the old model for cost/behavior consistency. The `claude agents` command gained 8 new configuration flags (`--add-dir`, `--settings`, `--mcp-config`, `--plugin-dir`, `--permission-mode`, `--model`, `--effort`, `--dangerously-skip-permissions`) for fine-grained background session control, replacing previous opaque defaults. Critical daemon fixes prevent freeze-on-sleep from clock jumps, crash-loops during binary upgrades, and Windows network-drive deadlocks. MCP timeout behavior now respects `MCP_TOOL_TIMEOUT` for remote HTTP/SSE servers (was capped at 60s).

**Why it matters:** Default model change impacts inference cost/speed for all background sessions. New agent flags enable enterprise-grade session orchestration but require CLI updates for automation scripts.

---

### **Agent SDK Breaking Changes: MCP Async Loading + Task API Migration**
[Python v0.2.82](https://github.com/anthropics/claude-agent-sdk-python/releases/tag/v0.2.82) | [TypeScript v0.3.142](https://github.com/anthropics/claude-agent-sdk-typescript/releases/tag/v0.3.142)

MCP servers now connect **asynchronously by default**—sessions start immediately with slow servers reporting `status: "pending"`. Set `MCP_CONNECTION_NONBLOCKING=0` to restore blocking 5s wait, or use `alwaysLoad: true` to require servers in turn 1. Task tools (`TaskCreate`/`TaskUpdate`/`TaskGet`/`TaskList`) replace deprecated `TodoWrite`—accumulate by task ID instead of snapshot replacement. TypeScript SDK also removed unstable v2 session API (`unstable_v2_createSession`, etc.)—migrate to `query()` with `AsyncIterable<SDKUserMessage>` for multi-turn or `options.resume` for continuation. Python SDK bumps `mcp` dependency to ≥1.23.0 to patch GHSA-9h52-p55h-vw2f (CVE-2025-66416) DNS rebinding vulnerability.

**Why it matters:** Async MCP loading improves startup latency but breaks workflows expecting synchronous server initialization. Task API migration is mandatory—`TodoWrite` is deprecated. Security patch is critical for production deployments.

---

### **TypeScript Agent SDK v0.3.143: Peer Dependency Shift**
[Release Notes](https://github.com/anthropics/claude-agent-sdk-typescript/releases/tag/v0.3.143)

`@anthropic-ai/sdk` and `@modelcontextprotocol/sdk` converted from dependencies to peerDependencies. Runtime unaffected (both are bundled), but **yarn classic users must manually install both packages** for full TypeScript type resolution. npm/bun/pnpm auto-install them, reducing duplicate installs across projects.

**Why it matters:** Yarn classic workflows break without manual `yarn add` for both packages. Migration reduces bundle bloat for multi-package monorepos.

---

### **GitHub Actions GA v1.0: Beta Deprecation + Enterprise Support**
[Docs](https://code.claude.com/docs/en/github-actions)

Claude Code GitHub Actions v1.0 requires workflow updates from beta: replace `@beta` with `@v1`, remove `mode` config (auto-detected), rename `direct_prompt→prompt`, consolidate CLI options into `claude_args` parameter. Claude Opus 4.7 available via `claude_args: --model claude-opus-4-7`. New enterprise features: Amazon Bedrock and Google Vertex AI integration with Workload Identity Federation for self-hosted deployments with data residency control.

**Why it matters:** Beta workflows stop working without migration. Enterprise users gain multi-cloud deployment options with Bedrock/Vertex support.

Plugin dependency enforcement prevents disabling plugins with dependents and auto-enables transitive dependencies—critical for complex plugin setups. ### **Claude Code v2.1.139: `/goal` Auto-Continue + Agent View**
[Release Notes](https://github.com/anthropics/claude-code/releases/tag/v2.1.139)

`/goal` command enables auto-continue across turns until completion with live token/turn tracking—simplifies complex multi-turn workflows without manual continue prompts. MCP stdio servers now receive `CLAUDE_PROJECT_DIR` environment variable, and `.mcp.json` edits apply without restart. 50+ fixes across transcript nav, scrolling, dialogs, and multi-image handling.

**Why it matters:** `/goal` reduces manual babysitting for long-running tasks. Unified agent view improves session monitoring at scale.
