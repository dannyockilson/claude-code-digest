# Claude Code Weekly Intelligence Digest

**This Week's Highlights**: Claude Code v2.1.38 stabilizes critical UX regressions affecting terminal navigation and Tab completion. The MCP ecosystem gains production-ready governance tools, persistent memory systems, and chaos testing infrastructure. TypeScript/Python SDKs now align with the newly-stable MCP spec 2025-11-25, bringing structured outputs and OAuth improvements.

---

## 🚨 Breaking Changes & Action Required

### [claude-code v2.1.38](https://github.com/anthropics/claude-code/releases/tag/v2.1.38)
Fixed VS Code terminal scroll-to-top regression from v2.1.37 and Tab key autocomplete behavior—critical for CLI workflows where Tab was incorrectly queueing slash commands instead of completing input. Also patches heredoc parsing security holes and blocks `.claude/skills` writes in sandbox mode.

### [claude-code v2.1.34](https://github.com/anthropics/claude-code/releases/tag/v2.1.34)
**Security fix**: Commands excluded from sandboxing could bypass Bash permission rules when `autoAllowBashIfSandboxed` was enabled. Now properly enforced. Also fixes crash when agent teams configuration changed between renders.

### [claude-code v2.1.32](https://github.com/anthropics/claude-code/releases/tag/v2.1.32)
Claude Opus 4.6 model now available. Experimental multi-agent collaboration via agent teams (opt-in with env var). Automatic memory recording/recall during sessions. Bash heredoc with template literals no longer breaks tool execution.

### [claude-code v2.1.30](https://github.com/anthropics/claude-code/releases/tag/v2.1.30)
PDF `pages` parameter enables reading specific ranges; large PDFs (>10 pages) now return lightweight references instead of full inlining. Fixed prompt cache invalidation bug—cache only cleared on tool name changes, not description/schema changes. 68% memory reduction for `--resume`.

### [claude-code v2.1.27](https://github.com/anthropics/claude-code/releases/tag/v2.1.27)
**Breaking**: Permissions now respect content-level `ask` over tool-level `allow`—`allow: ["Bash"], ask: ["Bash(rm *)"]` will now prompt instead of auto-executing destructive commands. Prevents privilege escalation via overly-broad tool grants.

### [anthropic-sdk-python v0.77.0](https://github.com/anthropics/anthropic-sdk-python/releases/tag/v0.77.0)
**BREAKING**: Structured Outputs API migration—replace `output_format` parameter with `output_config` in Messages API calls. Old parameter no longer supported.

### [anthropic-sdk-typescript sdk-v0.72.0](https://github.com/anthropics/anthropic-sdk-typescript/releases/tag/sdk-v0.72.0)
**Breaking Change**: Structured Outputs format migration from `output_format` to `output_config` parameter in Messages API. Existing implementations require code updates.

### [claude-agent-sdk-python v0.1.25](https://github.com/modelcontextprotocol/python-sdk/releases/tag/v1.25.0)
Branching strategy shift: `main` now tracks v2 development (breaking changes), while `v1.x` branch handles v1 maintenance. Pin to `mcp>=1.25,<2` to stay on v1. v2 release targeted for Q1 with major transport layer redesign.

### [typescript-sdk v1.26.0](https://github.com/modelcontextprotocol/typescript-sdk/releases/tag/v1.26.0)
**SECURITY**: Fixes critical vulnerability ([GHSA-345p-7cg4-v4c7](https://github.com/advisories/GHSA-345p-7cg4-v4c7)) where sharing server/transport instances could leak cross-client response data—upgrade immediately if you reuse MCP server instances.

### [claude-code-action v1](https://github.com/anthropics/claude-code-action/releases/tag/v1)
Claude Code GitHub Action v1.0 GA: Breaking API redesign consolidates 7+ deprecated inputs into unified `prompt` and `claude_args`. Migration required—`mode`/`direct_prompt`/`override_prompt` → `prompt`; `custom_instructions`/`model`/`allowed_tools` → `claude_args` format.

**Why it matters**: These permission and caching fixes directly impact security and performance in production Claude Code workflows.

### [MCP specification 2024-11-05](https://github.com/modelcontextprotocol/modelcontextprotocol/releases/tag/2024-11-05)
First formal release of the Model Context Protocol—establishes baseline specification for standardized context exchange between Claude and external tools/resources. Defines core protocol for tool discovery, resource access, and prompt integration patterns.

### [claude-context-os](https://github.com/Arkya-AI/claude-context-os)
A complete operating system for Claude that prevents context loss across sessions and enables reliable multi-session workflows. Positions itself as an OS layer managing session persistence, state management, and context continuity without relying on built-in conversation memory.

**Why it matters**: Solves the pain point of losing conversation history and state in complex, long-running projects.

### [claude-code-teams-mcp](https://github.com/cs50victor/claude-code-teams-mcp)
MCP server enabling Claude Code's agent teams orchestration to work with any external harness/framework. Bridges Claude Code's team coordination patterns for CI/CD, custom platforms, or alternative Claude interfaces.

### [MegaMemory](https://github.com/0xK3vin/MegaMemory)
MCP server adding persistent knowledge graph capabilities to Claude Code with semantic search and in-process embeddings. Enables agents to maintain context across sessions and navigate project structure intelligently without external API dependency.

**Why it matters**: Solves the stateless nature of standard Claude interactions with long-term project memory and semantic understanding.

### [autonomo](https://github.com/sebringj/autonomo)
MCP server adding cross-device orchestration to Claude Code—observe app state across multiple devices, control them simultaneously, and validate interactions within a single development session. Enables end-to-end testing and multi-device app development workflows.

### [deterministic-agent-control-protocol](https://github.com/elliot35/deterministic-agent-control-protocol)
Governance gateway for AI agents—bounded, auditable, session-aware control with MCP proxy, shell proxy, and HTTP API. Enforces policy guardrails across Cursor, Claude Code, and MCP-compatible tools. Shell proxy integration gates dangerous commands (rm -rf, etc.) at the protocol level.

**Why it matters**: Critical for teams needing deterministic, auditable AI-assisted development with compliance tracking.

### [XcodeMCPWrapper](https://github.com/SoundBlaster/XcodeMCPWrapper)
Bridges Xcode 26.3's non-compliant MCP implementation with Cursor and other strict MCP-spec clients by wrapping Xcode's MCP server to pass strict MCP specification validation.

### [mie](https://github.com/kraklabs/mie)
MCP server enabling persistent memory graphs for AI agents—stores facts, decisions, entities, and relationships across sessions/tools/providers. Works with Claude, Cursor, ChatGPT, and any MCP-compatible client. Solves context loss in multi-turn agent workflows.

### [HydraMCP](https://github.com/Pickle-Pixel/HydraMCP)
MCP server for multi-LLM orchestration from CLI—query GPT, Gemini, Claude, and local models simultaneously through existing subscriptions. Supports agent-to-agent communication patterns for comparing outputs, voting mechanisms, and synthesis workflows.

**Why it matters**: Eliminates vendor lock-in by standardizing LLM access through a single MCP interface.

### [MeiGen-AI-Design-MCP](https://github.com/jau123/MeiGen-AI-Design-MCP)
MCP server integrating local ComfyUI with Claude Code for AI-powered design generation. Includes 1,300+ curated prompt library and multi-directional parallel generation. Enables offline image generation workflows without cloud dependencies.
