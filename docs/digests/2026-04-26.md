## 🆕 New Releases

**Claude Code v2.1.119: Settings persistence + VCS expansion**  
[Release Notes](https://github.com/anthropics/claude-code/releases/tag/v2.1.119)  
Settings now persist to `~/.claude/settings.json` with precedence support for portable config. `--from-pr` expanded to GitLab/Bitbucket/GitHub Enterprise URLs. Fixes CRLF paste corruption, multi-line paste newline loss, vim Esc behavior, and slash command wrapping. OpenTelemetry now tracks `tool_use_id`, execution duration (`duration_ms`), and input size.  
*Why it matters:* Portable config eliminates per-project setup; enterprise VCS support critical for non-GitHub teams.

**Claude Code v2.1.118: Vim visual modes + MCP tool hooks**  
[Release Notes](https://github.com/anthropics/claude-code/releases/tag/v2.1.118)  
Vim visual modes (v/V) with selection and operators added for modal editing. Hooks can now invoke MCP tools directly via `type: "mcp_tool"` without subprocess overhead. Major MCP OAuth fixes resolve token refresh races, keychain corruption on macOS, missing `expires_in` handling, and scope re-consent logic.  
*Why it matters:* Hooks invoking MCP tools = automation without bash scripts; OAuth fixes eliminate production authentication failures.

**Claude Code v2.1.117: Embedded search tools + OAuth refresh**  
[Release Notes](https://github.com/anthropics/claude-code/releases/tag/v2.1.117)  
Native macOS/Linux builds now embed `bfs`/`ugrep` in Bash tool, eliminating subprocess round-trips for glob/grep operations. OAuth token refresh now triggers on 401 for Plain-CLI, fixing mid-session auth expiration. `/model` selection persists across restarts; startup header clarifies source (project vs managed-settings).  
*Why it matters:* Faster file searches without tool fragmentation; no more sessions dying when tokens expire.

**Claude Code v2.1.116: 67% faster session resume + terminal UX fixes**  
[Release Notes](https://github.com/anthropics/claude-code/releases/tag/v2.1.116)  
`/resume` now 67% faster on 40MB+ sessions with improved dead-fork handling. MCP startup accelerated by deferring `resources/templates/list` until first @-mention. Terminal improvements: fullscreen scrolling via `/terminal-setup`, inline thinking progress, Ctrl+- undo, and Cmd+Left/Right navigation in Kitty. Sandbox auto-allow now enforces dangerous-path checks for `rm`/`rmdir`.  
*Why it matters:* Large conversation histories no longer bottleneck productivity; terminal UX now matches modern editors.

**Claude Code GitHub Action v1.0.107 GA**  
[Release Notes](https://github.com/anthropics/claude-code-action/releases/tag/v1.0.107)  
Official GA release enables automated code generation and analysis in GitHub CI/CD pipelines. Claude can read/modify files, run commands, and create pull requests directly in Actions. Use `anthropics/claude-code-action@v1.0.107` in workflow YAMLs for production deployments.  
*Why it matters:* Claude in CI/CD = automated code reviews, test generation, and PR creation without manual intervention.

## 🔌 MCP Ecosystem

**Selvedge: Git blame for AI agent reasoning**  
[GitHub](https://github.com/masondelan/selvedge)  
MCP server that captures AI agent reasoning and decision-making during code changes, storing rationale in local SQLite. Git blame shows *what* and *who*, Selvedge records *why*—the agent's reasoning context at commit time. Zero dependencies, local storage. Designed for Claude Code workflows where understanding past AI decisions matters for future edits.  
*Why it matters:* Persistent agent intent solves the "why did Claude do this?" problem across sessions.

**DecisionGraph: Engineering decision memory from GitHub/Slack/Jira**  
[GitHub](https://github.com/hieuchaydi/DecisionGraph)  
MCP-enabled system that integrates GitHub/Slack/Jira evidence to answer architectural 'why' questions and enforce pre-change guardrails via CLI/API. Captures decision provenance across distributed communication channels.  
*Why it matters:* Prevents regressions by surfacing past architectural decisions when Claude proposes changes.

**Guild: Local-first cognition substrate for agents**  
[GitHub](https://github.com/mathomhaus/guild)  
Single Go binary + SQLite for persistent memory/knowledge with hybrid keyword+semantic search. MCP-compatible substrate for agents needing stateful context beyond single conversations. Enables long-term agent reasoning without external APIs or databases.  
*Why it matters:* Local-first architecture = privacy + zero API dependencies for agent memory.

**pycentauri: MCP server for Elegoo 3D printers**  
[GitHub](https://github.com/bjan/pycentauri)  
Python client, CLI, and MCP server for Elegoo Centauri Carbon 3D printers. Enables AI-assisted printer control and monitoring through Claude's native MCP protocol for print job management and diagnostics.

**agent-task-mcp-server: Multiuser kanban for agents**  
[GitHub](https://github.com/dizlexic/agent-task-mcp-server)  
MCP server enabling multiuser task management through kanban board interface. Claude instances read/write tasks across multiple boards with persistence. Solves shared state management for multi-step workflows and team task coordination without external databases.

**tnl: Structured English contracts for agents**  
[GitHub](https://github.com/janaraj/tnl)  
Enables AI coding agents to create and enforce structured English contracts that persist across sessions. Agents propose contracts, developers approve, and future sessions read/follow these agreements. Maintains consistent agent behavior and developer intent without code changes.

**anthropic-sdk-typescript v0.91.0–0.91.1: CMA Memory + security fix**  
[v0.91.0](https://github.com/anthropics/anthropic-sdk-typescript/releases/tag/sdk-v0.91.0) | [v0.91.1](https://github.com/anthropics/anthropic-sdk-typescript/releases/tag/sdk-v0.91.1)  
CMA Memory public beta in TypeScript SDK v0.91.0. v0.91.1 adds security fix: memory files now use restrictive file mode to prevent unauthorized access. Bedrock auth header support for mantle client improves AWS integration reliability.  
*Why it matters:* Security fix is critical if you're persisting memory data; upgrade immediately.

**claude-agent-sdk-python v0.1.67: Trio async runtime fix**  
[Release Notes](https://github.com/anthropics/claude-agent-sdk-python/releases/tag/v0.1.67)  
Trio async runtime compatibility restored after regression in v0.1.51. Fixes `RuntimeError: no running event loop` for trio users via sniffio-based event loop detection. New explicit dependency: sniffio>=1.0.0. Bundled Claude CLI updated to 2.1.120.

**claude-agent-sdk-python v0.1.65: Session performance + thinking display**  
[Release Notes](https://github.com/anthropics/claude-agent-sdk-python/releases/tag/v0.1.65)  
`list_session_summaries()` reduces N-session fetches from N round-trips to 1 via O(1) summary sidecars. New `import_session_to_store()` enables local→remote session migration. Server tool execution now visible—fixed silent drops of `server_tool_use` and `advisor_tool_result` blocks. Thinking display control added via `ThinkingConfig.display`, overriding Opus 4.7's `"omitted"` default.  
*Why it matters:* N→1 round-trips = massive speedup for multi-session management; thinking display fixes Opus 4.7 silent reasoning.

**claude-agent-sdk-python v0.1.64: SessionStore adapters**  
[Release Notes](https://github.com/anthropics/claude-agent-sdk-python/releases/tag/v0.1.64)  
SessionStore protocol + InMemorySessionStore implementation enables persistent multi-turn session management. Three production-ready reference adapters (S3, Redis, Postgres) available as copy-in examples. Conformance test harness at `claude_agent_sdk.testing.run_session_store_conformance` simplifies third-party adapter validation. Adds `--session-mirror` CLI flag and 9 async helper functions for session resume/fork/list.  
*Why it matters:* Production persistence is now trivial; copy-paste S3/Redis/Postgres adapters instead of rolling your own.

**claude-agent-sdk-typescript v0.2.119: Prompt caching + reconnection**  
[Release Notes](https://github.com/anthropics/claude-agent-sdk-typescript/releases/tag/v0.2.119)  
`excludeDynamicSections` separates static auto-memory instructions into cacheable system-prompt block, moving only per-user memory paths and per-machine env values to first user message—improves prompt caching efficiency. Long-running sessions now auto-reconnect claude.ai-proxied MCP servers after transport-stream aborts. `SessionStore.append()` retries up to 3 times with backoff before dropping batches.  
*Why it matters:* Prompt caching optimization = lower costs on repeated sessions; auto-reconnect prevents agent failures.
