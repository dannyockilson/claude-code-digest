## 🚨 Breaking Changes & Action Required

### **Claude Code v2.1.92**
[Release Notes](https://github.com/anthropics/claude-code/releases/tag/v2.1.92)

Added `forceRemoteSettingsRefresh` policy setting for security-critical environments with fail-closed enforcement of managed settings. Interactive Bedrock setup wizard simplifies 3rd-party platform authentication (AWS region/credentials/model pinning). Fixed critical subagent spawning bug that permanently failed after tmux window kills/renumbering in long-running sessions.

**Why it matters:** Enterprises with centrally-managed Claude Code deployments can now enforce settings distribution without silent fallbacks.

### **Claude Code v2.1.90**
[Release Notes](https://github.com/anthropics/claude-code/releases/tag/v2.1.90)

Fixed critical `--resume` regression causing full prompt-cache miss on first request (v2.1.69+), impacting cache efficiency for users with deferred tools/MCP servers. Hardened PowerShell tool permissions: blocked background job bypass (`&`), debugger hangs, TOCTOU vulnerabilities, and added DNS cache queries to deny-list.

**Why it matters:** Restores cache efficiency for headless automation workflows and closes privilege escalation vectors in PowerShell tool.

### **Claude Code v2.1.89**
[Release Notes](https://github.com/anthropics/claude-code/releases/tag/v2.1.89)

**Breaking: `defer` permission workflow** — PreToolUse hooks can now pause tool execution and resume with `-p --resume` for re-evaluation; PermissionDenied hook added for auto-mode classifier denials with retry capability. **Performance fix: MCP connection handling** — `MCP_CONNECTION_NONBLOCKING=true` skips MCP wait in `-p` mode; server connections now timeout at 5s instead of blocking.

**Why it matters:** Enables headless automation to skip blocking MCP connections and adds retry logic for permission denials—critical for CI/CD.

### **MCP TypeScript SDK @modelcontextprotocol/server@2.0.0-alpha.1**
[Release Notes](https://github.com/modelcontextprotocol/typescript-sdk/releases/tag/%40modelcontextprotocol/server%402.0.0-alpha.1)

**Breaking: Error handling** — Unknown tool calls now return JSON-RPC errors (code -32602) instead of CallToolResult.isError; unknown resources return -32002 (ResourceNotFound). Callers must catch rejected promises. **Breaking: Task config relocation** — taskStore, taskMessageQueue, defaultTaskPollInterval, and maxTaskQueueSize moved from ProtocolOptions to capabilities.tasks. **Breaking: Standard Schema support** — Tool/prompt schemas now accept any Standard Schema library (Zod v4, Valibot, ArkType) instead of Zod-only; removed schema utility exports.

**Why it matters:** All TypeScript MCP server implementations must update error handling and config structure. Standard Schema support allows dropping Zod as a peer dependency.

### **@modelcontextprotocol/node@2.0.0-alpha.1**
[Release Notes](https://github.com/modelcontextprotocol/typescript-sdk/releases/tag/%40modelcontextprotocol/node%402.0.0-alpha.1)

Deprecated `.tool`, `.prompt`, `.resource` method signatures removed—update server implementations to new method signatures. Hono now respects `overrideGlobalObjects: false`, preventing Response object conflicts with Next.js. Project standardized on pnpm exclusively.

**Why it matters:** Breaking method signature changes require MCP Node server rewrites; Next.js users can now safely use Hono transport.

### **Agent SDK TypeScript v0.2.91**
[Release Notes](https://github.com/anthropics/claude-agent-sdk-typescript/releases/tag/v0.2.91)

**BREAKING:** Sandbox now defaults `failIfUnavailable: true` — queries will error and exit if sandbox deps missing instead of silently degrading; set `failIfUnavailable: false` to restore old behavior. Added `terminal_reason` field to result messages exposing query loop exit conditions.

**Why it matters:** Silent degradation is now opt-in; production code must handle sandbox unavailability explicitly.

## 📄 Docs Changes

### **Claude Code Overview**
[Docs](https://code.claude.com/docs/en/overview)

Major docs overhaul: Terminal CLI now prominently featured with native install, Homebrew, and WinGet methods. MCP integration highlighted as first-class feature. Multi-surface session continuity emphasized (Remote Control, Dispatch, --teleport, /desktop hand-off). Agent teams and Agent SDK documented as core capabilities.

**Why it matters:** MCP and CLI automation now positioned as primary workflows, not secondary features.

### **Claude Code Hooks Guide**
[Docs](https://code.claude.com/docs/en/hooks-guide)

New Hooks documentation: lifecycle automation at SessionStart, PreToolUse, PostToolUse, PermissionRequest, etc. Configured in `~/.claude/settings.json`. Use `/hooks` CLI to browse configured hooks and event counts.

**Why it matters:** Auto-format code post-edit, block protected files, inject context after compaction, reload env vars on directory change—all via declarative hooks.

### **Claude Code MCP**
[Docs](https://code.claude.com/docs/en/mcp)

MCP servers now support dynamic tool updates via `list_changed` notifications. Plugin-bundled MCP servers (`.mcp.json` in plugins) auto-start when plugins are enabled. Three-tier MCP scope system (local/project/user) with clear precedence rules. Push-message channels (`claude/channel` capability + `--channels` flag) enable MCP servers to react to external events.

**Why it matters:** Project-scoped MCP configs enable team collaboration via version control; dynamic tool updates eliminate reconnection overhead.
