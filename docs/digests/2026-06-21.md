## 🚨 Breaking Changes & Action Required

### Claude Code v2.1.183: Auto Mode Safety Hardening
[Release Notes](https://github.com/anthropics/claude-code/releases/tag/v2.1.183)

Auto mode now blocks destructive git/infrastructure commands (`git reset --hard`, `git stash drop`, `terraform destroy`, `rm -rf`, etc.) unless explicitly requested. Deprecated model warnings surface in stderr.

**Why it matters:** If you rely on auto mode for destructive operations, you'll need to explicitly request them or adjust workflows to avoid blocked commands.

---

### MCP Python SDK v2.0.0a2: Strict Protocol Validation
[Release Notes](https://github.com/modelcontextprotocol/python-sdk/releases/tag/v2.0.0a2)

Per-version protocol validation now enforces spec compliance: inbound requests validate against negotiated version (rejecting invalid payloads with `METHOD_NOT_FOUND`/`INVALID_PARAMS`), outbound responses strip newer-version fields for backward compatibility. Handlers returning spec-invalid output (e.g., `Tool(inputSchema={})` without `"type": "object"`) now fail with `INTERNAL_ERROR` instead of silently passing. ClientSession rewritten on JSONRPCDispatcher—server-initiated requests (sampling, elicitation) run concurrently instead of blocking, fixing deadlocks.

**Why it matters:** Stricter validation than v2.0.0a1. Opt-in only (`pip install mcp==2.0.0a2`)—v1.x remains default. Test your MCP servers before upgrading; invalid schemas that previously passed will now fail.

---

### MCP Python SDK v1.28.0: WebSocket & Tasks API Deprecation Warnings
[Release Notes](https://github.com/modelcontextprotocol/python-sdk/releases/tag/v1.28.0)

WebSocket transport (`mcp.client.websocket`, `mcp.server.websocket`) and experimental tasks API emit `DeprecationWarning`—both removed in v2. Migrate to streamable HTTP transport now.

**Why it matters:** If you're using WebSocket or `ClientSession.experimental`, you have one major version to migrate before breakage.

---

### anthropic-sdk-python v0.109.2 & TypeScript v0.104.2: Retired Models Removed
[Python](https://github.com/anthropics/anthropic-sdk-python/releases/tag/v0.109.2) | [TypeScript](https://github.com/anthropics/anthropic-sdk-typescript/releases/tag/sdk-v0.104.2)

Deprecated model identifiers removed from both SDKs. Check your code for hardcoded model strings referencing older Claude versions.

**Why it matters:** Code specifying retired model IDs will break. Update to current model names before upgrading SDKs.

## 🆕 New Releases

### Claude Code v2.1.181: Unified Config Syntax & Critical Bug Fixes
[Release Notes](https://github.com/anthropics/claude-code/releases/tag/v2.1.181)

`/config key=value` syntax now sets any option (thinking, model, etc.) from the prompt across all modes, eliminating mode-specific commands. Fixed critical prompt caching bug affecting custom `ANTHROPIC_BASE_URL` and Foundry users (per-request attestation tokens now properly cached). Upgraded bundled Bun to 1.4 with improved streaming for long paragraphs (line-by-line rendering). Fixed startup regressions: 120ms delay in fresh environments, 15s blocking on slow network settings fetch, TUI freezing on macOS with busy Spotlight.

**Why it matters:** Streamlines workflow configuration and eliminates multiple blocking startup issues on macOS.

**Why it matters:** Major UX regression fix—stuck spinners and dropped responses were breaking multi-turn workflows.

### Claude Code v2.1.178: Fine-Grained Tool Permissions & Nested Skills
[Release Notes](https://github.com/anthropics/claude-code/releases/tag/v2.1.178)

`Tool(param:value)` syntax enables permission rules filtered by input parameters (e.g., `Agent(model:opus)` blocks Opus subagents). Nested `.claude/` directories now load skills/agents/workflows closest to working directory; name collisions resolved with `<dir>:<name>` syntax. Auto mode security improved: subagent spawns classified before execution. Fixed critical bugs: subagent permission evaluation before launch, Remote Control connection status now persistent in footer, MCP server-level specs in `disallowedTools` honored.

**Why it matters:** Fine-grained permission control enables blocking specific model usage or tool parameters without blanket denials—critical for multi-project workspaces with varying security requirements.
