## 🚨 Breaking Changes & Action Required

**Claude Code v2.1.94–v2.1.98: Default behavior & auth changes**  
v2.1.94 raises default effort level to 'high' for API-key/Bedrock/Vertex users, increasing resource consumption—control via `/effort` command. v2.1.96 fixed critical Bedrock auth regression breaking `AWS_BEARER_TOKEN_BEDROCK` and `CLAUDE_CODE_SKIP_BEDROCK_AUTH` (403 errors). v2.1.98 patches three security bypasses: bash permission bypass via backslash-escaped flags, compound commands bypassing forced prompts, and env-var prefixes not prompting. Also adds Perforce integration via `CLAUDE_CODE_PERFORCE_MODE` to prevent silent overwrites on read-only files.  
**Why it matters:** Update to v2.1.98+ immediately if using Bedrock auth or sandboxed execution; review effort level impact on API costs.

**claude-code-action v1.0.90: MCP timeout config now enforced**  
MCP_TIMEOUT, MCP_TOOL_TIMEOUT, MAX_MCP_OUTPUT_TOKENS now properly forwarded to action steps, fixing silent timeout failures in CI/CD. Security fix rejects PATH_TO_CLAUDE_CODE_EXECUTABLE with control characters to prevent injection attacks.  
**Why it matters:** Existing GitHub Actions workflows may now surface previously hidden MCP timeout errors.

Fixed critical security vulnerability in POSIX `which` fallback used by LSP binary detection (command injection risk). Major permission system hardening fixes silent downgrades and mid-session setting changes. Extensive NO_FLICKER stability: memory leaks, rendering artifacts in zellij/Windows Terminal, CJK text corruption, mouse-wheel performance. Fixed critical MCP issues: HTTP/SSE buffer leaks (~50MB/hr), OAuth token refresh with ADFS, 429 exponential backoff, subagent working directory leaks.  
[Release notes](https://github.com/anthropics/claude-code/releases/tag/v2.1.97)

## 🔧 Tools & Plugins

**stacklit: 250-token codebase summaries replace 50K+ token exploration**  
Auto-generates semantic codebase context via single command, cutting token bloat by ~200x. Auto-configures Claude Code, Cursor, and Aider without manual setup. MCP-compatible for faster, cheaper agent interactions with large codebases.  
**Why it matters:** Solves token window waste during agentic workflows—particularly valuable for Pro/Max tier token budgets.  
[GitHub](https://github.com/glincker/stacklit)

**mempalace: Local semantic search over conversation history**  
Stores all AI conversation history locally using ChromaDB with 96.6% LongMemEval R@5 accuracy. Integrates with Claude Code via MCP with 19 tools—after one-time setup, Claude automatically searches your conversation archive. Runs entirely local with zero API calls; supports code/docs/conversation imports across Claude, ChatGPT, Slack.  
**Why it matters:** Eliminates context re-establishment between sessions without summarization loss; authors publicly corrected overstated launch claims (AAAK compression regresses vs raw mode).  
[GitHub](https://github.com/MemPalace/mempalace)

**llm-wiki-kit: Persistent, self-maintaining knowledge bases**  
Implements Karpathy's LLM Wiki pattern—drop PDFs/URLs/videos once, agent retains context across sessions without re-explaining. Compounds research over time, reducing prompt overhead and context bloat for long-running projects.  
[GitHub](https://github.com/iamsashank09/llm-wiki-kit)

**memctx: Autonomous session memory for Claude Code**  
Adds persistent session memory with AI-powered summaries and searchable history. Automatically injecting relevant past interactions eliminates manual context re-establishment and reduces prompt overhead during long-running projects.  
[GitHub](https://github.com/bbhunterpk-ux/memctx)

**logseq-cli: Unix-composable CLI for Logseq**  
New Unix-style CLI tool for Logseq's local HTTP API, enabling shell composition and piping workflows. Designed for both humans and AI agents—composable with grep/jq for knowledge management automation.  
[GitHub](https://github.com/wolf-jonathan/logseq-cli)

## 🔌 MCP Ecosystem

**Plugin-bundled MCP servers (Claude Code MCP docs update)**  
Plugins can now package MCP servers in .mcp.json, auto-starting with plugins and eliminating manual MCP configuration. Three-tier scoping (local/project/user) with precedence hierarchy and environment variable expansion enables shared team configs while preserving sensitive credentials per-machine. Dynamic tool updates via list_changed notifications—servers can update capabilities without reconnection.  
**Why it matters:** Dramatically simplifies MCP server distribution and team onboarding.  
[Docs](https://code.claude.com/docs/en/mcp)

**MCP Specification security update**  
Expanded security framework emphasizing user consent, data privacy, tool safety, and LLM sampling controls. Key addition: tool descriptions are untrusted unless from verified servers, requiring hosts to obtain user consent before any tool invocation. New "Elicitation" feature enables server-initiated user information requests.  
[Specification](https://modelcontextprotocol.io/specification)

**Chiasmus: Formal verification MCP server**  
Enables Claude to access formal verification tools, allowing verification of mathematical proofs and program correctness directly within conversations. Bridges gap between LLM reasoning and rigorous formal verification—useful for safety-critical systems.

**mxLore: Self-hosted knowledge base MCP server**  
Self-hosted MCP server adding persistent knowledge base with 39 tools and 11 skills for maintaining institutional memory across coding sessions. No data leaves your infrastructure—critical for enterprises.

**google-meta-ads-ga4-mcp: Marketing platform MCP server**

**agent-inbox: Disposable email for agents**  
MCP server enabling AI agents to generate disposable email inboxes via single tool call—useful for email verification, auth flow testing, and automated workflows without external dependencies.  
[GitHub](https://github.com/gsd-build/agent-inbox)

**Astravue MCP server: Project management integration**  
Official MCP server enabling AI-driven management of Astravue projects, tasks, time tracking, and teams through Claude Code. Integrates project/task management directly into Claude workflows.

## 🔬 Research Queue

**Microsoft Memento: Extending context via block-based reasoning**  
Extends LLM context windows by splitting chain-of-thought reasoning into blocks with summaries, then evicting processed blocks from KV cache. Provides data pipeline (converts CoT traces to Memento format) + vLLM overlay with KV cache block masking. Ready-to-use: pip install vllm 0.13.0, run data pipeline, deploy via OpenAI-compatible API server. Research-backed with OpenMementos dataset; MIT licensed, but requires careful prompt engineering with XML-style delimiters.  
**Why it matters:** Enables longer reasoning within fixed context limits—relevant for Claude Code sessions hitting context boundaries.  
[GitHub](https://github.com/microsoft/memento)
